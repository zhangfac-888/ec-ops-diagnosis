# 电商运营诊断 Skill v1.5.0

> 一个入口走完电商运营全链路诊断：**主图 → 详情页 → 人群 → 数据 → 运营策略**。
>
> 核心约束：**没有真实数据不硬给结论，全程不编百分比、不预测业绩、来源必标。**

---

## 这个 Skill 解决什么问题

做电商，五件事本来要分别找五套方法、还容易被 AI 用"看着对其实没数据"的纸面分析糊弄：

1. 主图行不行？ 2. 详情页哪屏拖后腿？ 3. 这物料谁会买、谁会卡？ 4. 一堆流量数据说明啥？ 5. 接下来到底先打哪个？

本 Skill 把这五件事串成**一条带统一纪律的诊断流水线**：每段都强制"真实数据进、可溯源结论出"，缺数据就降级标注而不是瞎编，且**前一段的诊断直接喂给后一段**（如人群验证挖出的"最致命障碍"直接交给详情页去改）。

## 安装

### ⚡ npx 一句话安装（推荐 · 跨平台 · 需 Node ≥16.7）

装了 Node 的话，一条命令搞定，Windows/mac/Linux 通用：

```
npx ec-ops-diagnosis
```

> 还没发布到 npm 时，可直接从 GitHub 装（效果相同）：
> ```
> npx github:zhangfac-888/ec-ops-diagnosis
> ```

它会把 skill 装进 `~/.claude/skills/电商运营诊断/`（覆盖式安装，自动清掉旧版本残留文件）。加 `--dry-run` 可先预览要装哪些文件。

### 🚀 一行命令安装（无 Node 也能用）

复制一条命令，直接下载并装进 Claude Code 的 skills 目录：

- **macOS / Linux**：
  ```
  curl -fsSL https://raw.githubusercontent.com/zhangfac-888/ec-ops-diagnosis/main/install-remote.sh | bash
  ```
- **Windows（PowerShell）**：
  ```
  irm https://raw.githubusercontent.com/zhangfac-888/ec-ops-diagnosis/main/install-remote.ps1 | iex
  ```

装完重启 Claude Code，说「跑电商运营诊断」即可。

### 本地脚本安装（已下载 zip / 已 clone 后）

在解压或克隆后的文件夹里运行：
- **Windows**：`powershell -ExecutionPolicy Bypass -File .\install.ps1`
- **macOS / Linux**：`bash install.sh`

### 手动安装（Claude Code）

1. 解压整个文件夹到 `~/.claude/skills/电商运营诊断/`（保持 `子技能/`、`references/` 子目录结构别拆散）。
2. 重启 Claude Code。触发词：「电商运营诊断 / 给我的产品做体检 / 主图行不行 / 详情页诊断 / 谁会买 / 这堆数据怎么读 / 接下来打什么」。

### Kimi / 其他 AI 助手

1. 解压文件夹。
2. 把 `SKILL.md` 全文贴给 Kimi 当系统提示 / 或上传 `SKILL.md` 文件。
3. 走到某一段时，**把路由表里对应的那个文件一起上传/贴入**（如做人群验证就上传 `子技能/人群验证引擎/SKILL.md` + `原型卡片/` 里相关卡片）。Kimi 没有自动读本地文件夹的能力，需要你手动把要用的那段喂给它。

### Claude Desktop / 网页版

同上：`SKILL.md` 当系统提示，需要哪段就把对应 references / 子技能文件粘进去。

## 怎么用

### 单跑一段（最常见）

```
帮我诊断这张主图：
- 品类：家用胶囊咖啡机（699 元）
- 人群：25-35 岁一二线租房白领
- 竞品：N 喊"咖啡馆品质"／D 喊"无线便携"
[贴主图]
```

### 整链体检（一个店从头到尾）

```
给我的店做整链体检：
1) 主图和详情页截图在下方 2) 真实客群占比(带来源) 3) 流量/客群/SKU 三张表
[依次贴上]
```
→ 会按 主图→详情页→人群→数据→运营策略 依次跑，前段结论喂后段，最后出一份**按优先级排好、每条标来源**的运营动作清单。

> ⚠️ **发截图/文字，别发链接**——天猫/淘宝反爬，AI 抓不到只会编。

## 核心能力

✅ 五段诊断串成一条链，前段产物自动喂后段
✅ 五段全程统一纪律：缺数据降级标注 / 不编百分比 / 不预测业绩 / 来源必标 [数据]/[推断]
✅ 人群验证内置八大人群原型卡片（覆盖素材库），按卡片真实代入、不靠名字印象
✅ 数据分析用确定性引擎算 flag（黑洞渠道/死流量人群/王者SKU），AI 只加定性解读不碰数字
✅ 每段附「数据完备度」评分 + 「判断不了的」清单
✅ 第⓪段真实数据采集：教你从评论/后台拿真数据并洗干净再喂后面（红线⑤要的真实客群构成有了着落）
✅ 渠道与人群速查：平台气质矩阵 + 购买决策驱动 + 平台合规禁用词
✅ 每段配「📋 可粘贴提示词」，豆包/Kimi/ChatGPT 不装 skill 也能用
✅ 带 agents/openai.yaml，可上传 Codex/扣子 一句话唤起
✅ 自包含、零外部依赖、零抓取、跨平台通用
✅ **自进化（判例制）**：每次诊断把"是非判例"存进经验库，靠你回填真实结果升级（[推断]→[数据]），越用越准——但绝不沉淀频率/预测
✅ **复诊收敛**：改完带"改了什么"回来，只诊断 delta、判最致命障碍消解没，收敛即停、最多 3 轮不空转

## 这个 Skill 不做的事（边界）

❌ 不自动抓详情页/主图链接（反爬+登录墙）
❌ 不预测转化率/涨幅/GMV（模型没数据支撑）
❌ 不替你做市场调研、不替你编客群占比（真实数据必须你给）
❌ 不依赖任何外部 MCP、账号、登录态或网页服务

## 常见问题（FAQ）

**Q：为什么不让我发商品链接，要我截图/贴文字？**
天猫/淘宝/京东都有反爬+登录墙，AI 跨用户抓不到你的页面，硬抓只会编造产品信息。截图几秒钟，结果可靠。

**Q：为什么不给我"改完能涨多少转化率/销量"？**
那是模型没有数据支撑的预言。本诊断只给站得住的判断和方向——这正是它跟"什么都敢承诺"的工具的区别。

**Q：我没有真实客群占比/数据，能跑吗？**
能，但会**降级**：人群段只列障碍不排序、并标"排序不可信"。想要可信排序，先按 `references/真实数据采集.md`（第⓪段）把真数据取到手。模型不替你编人口构成。

**Q：这个文件夹安全吗？为什么 Windows 可能提示"未知发布者"？**
装进 skill 目录的内容是纯文本方法论（markdown + txt + yaml）。包里另有几个**透明、可自行打开审阅**的安装脚本（`install*.sh/ps1`、`bin/cli.js`），它们只做一件事：把这些文本文件复制到 `~/.claude/skills/` 下，不联网回传、不读你的其它文件。建议从可信来源获取，并在运行前自行过目脚本内容。

**Q：它会"越用越准"吗？我的数据会被它学走吗？**
会越用越准，但方式很克制：它只把**是非判断**（如"某障碍对某人群成立"）存进 `data/经验库.md`，且只有你回填**真实结果**才会把它升级成可信结论；它**绝不**沉淀"X% 的人会怎样"这类频率、也不存涨幅预测。经验库就在你本地这个文件夹里，不上传任何地方。
