# 电商运营诊断（总编排）

> 一台把「四诊断 ⟶ 运营策略」串成链的总台。你给一个产品 + 物料 + 真实数据，它按统一纪律带你走完整条诊断链，输出**可溯源、缺数据降级、不编不预测**的诊断与动作。
>
> 世界观：**真实数据至上**。模型擅长"这条障碍对这群人成不成立 / 这屏结构对不对 / 这个渠道是不是黑洞"（是非·分级·算术题），不擅长"百分之多少的人会怎样 / 改完涨多少"（频率·预测题）。本 Skill 只产前者，焊死不产后者。
>
> 📌 **出身**：本方法论经真实电商店铺整链实战检验、反复打磨——不是纸面框架。别的工具敢拍胸脯给你涨幅，本诊断只给**站得住的判断**。

## 🛠️ 在哪用 · 最快上手

- **Claude Code / Codex / 扣子**：装成 skill 调用（见 README）；Codex 可用 `agents/openai.yaml` 一句话唤起。
- **豆包 / Kimi / ChatGPT 等普通 AI**：不装 skill 也能用——走到哪段，就把那段 references 里的「📋 可粘贴提示词」连同你的物料一起发给它。
- **第一次没真实数据？** 别空跑：先按 `references/真实数据采集.md`（第⓪段）把真数据取到手再进诊断。模型不替你编。

## 🎯 这个 Skill 能做什么

| 你给我什么 | 我带你走哪段 |
|----------|------------|
| 一堆评论 / 后台报表 | **第⓪段·真实数据采集**：洗成客群占比/痛点/三联表，喂给后面各段 |
| 一张主图 + 情境 | **主图诊断**：这张图在这个情境下行不行、改哪 |
| 整套详情页截图 + 情境 | **详情页诊断**：五维审一遍、哪屏拖后腿 |
| 物料 + 真实客群构成 | **人群验证**：谁会卡、卡在哪、哪条障碍最致命 |
| 流量/客群/SKU 三张表 | **数据分析**：黑洞渠道 / 死流量人群 / 王者SKU 在哪 |
| 上面几段的诊断结论 | **运营策略**：按优先级出动作，每条标来源 |
| 一个店从头到尾 | **整链体检**：五段依次跑，前段喂后段 |

可整链走，也可单跑某一段（数据契约满足即可）。

## 🚦 触发

用户说「跑电商运营诊断」「给我这产品/店铺做个体检」「我这主图/详情页行不行」「谁会买」「这堆数据怎么读」「接下来该打什么」——进本 Skill，先判要走哪段（见路由表），再按对应方法论走。

## 🟥 贯穿红线（五段全程遵守，违任一返工）

1. **缺数据降级标注**——数据契约缺一不硬给结论，按降级路径走、并红标说明。
2. **不编百分比/频率**——「X% 的人会划走 / 命中率 30%」一律禁。频率题模型有效性≈0。
3. **不预测业绩**——CTR / 转化率 / 涨幅 / 销量 / GMV 任何数字预言一律禁；只给方向。
4. **来源必标**——每条结论挂 `[数据]` / `[推断]` / `basis`（哪几条情境/哪份诊断在起作用），可溯源。
5. **不自抓链接**——天猫/淘宝反爬抓不到；要诊断 → 发截图/文字。模型不替你抓、不替你编人口构成。

> 产出前自己过一遍这五条，命中即改。

## 📋 数据契约（开工门槛）

| 段 | 必填 | 缺了怎么办 |
|----|------|-----------|
| 通用（每段都要） | 品类 / 客单价 / 人群 / 参数 / 竞品（+渠道） | 对应维度判断标 `[推断]`、basis 留空并注明"情境缺失"，不硬给 |
| 人群验证 额外 | 真实客群构成（人群占比 + 来源） | **降级**：只列障碍、全程"排序不可信"红标，不排序 |
| 数据分析 | 流量结构表 / 客群错位表 / SKU 矩阵表（手填真实数） | 哪张表空，该维度不算，不编数字 |

**数据完备度评分**：每段输出末尾附「数据完备度 x/10」。满分 10，每缺一个通用维度 −1.5，无视觉能力 −2。

## 🔗 链路与 handoff

```
⓪真实数据采集(评论/后台 → 客群占比·痛点·三联表)
        │ 喂真实输入(红线⑤要的真实客群构成就来自这)
        ▼
主图诊断 ─→ 详情页诊断 ─→ 人群验证 ──最致命障碍──▶ 详情页"怎么改"(手术)
                            ▲
            数据分析 ──客群错位──┘ (指导该代入哪些有份量的人群)
                │
            三份摘要(详情页 + 人群 + 数据) ──▶ 运营策略(按优先级出动作)
```

- **整链走**：主图→详情页→人群→数据→策略，前段产物喂后段。
- 关键接力：
  - 人群验证产「最致命障碍」→ 喂详情页"怎么改"，省掉"拍人群"翻车。
  - 数据分析识别「客群错位」（访问端 vs 支付端不一致）→ 指导人群验证该代入谁。
  - 运营策略自动消费 ①②③ 三份最新诊断摘要；缺哪份标"未做"、不基于它下结论。

## 🧭 路由表（每段：何时进 · 方法论在包内哪个文件）

| 段 | 何时进 | 方法论正文（包内相对路径） | 一句话纪律 |
|----|-------|--------------------------|-----------|
| ⓪ 真实数据采集 | 手里只有评论/报表、没现成真实数据 | `references/真实数据采集.md` | 真实计数可用；样本不代表盘子要标；不替你编 |
| ① 主图诊断 | 只看单张主图好不好 | `references/主图诊断.md` | 三层判断；好坏标 basis；缺情境→[推断]；禁 CTR 预测 |
| ② 详情页诊断 | 审整套详情页 / 详情页怎么改 | `子技能/详情页架构师/SKILL.md` | 审五维(结构/排序/视觉/因果/转化)；basis 取6维；禁涨幅预测 |
| ③ 人群验证 | 想知道"谁会卡、哪条障碍最致命" | `子技能/人群验证引擎/SKILL.md` | 8红线；严重度四档；每障碍带"❗判断不了的"；禁频率%；**必实际读卡片代入** |
| ④ 数据分析 | 有流量/客群/SKU 真实数据要读 | `references/数据分析.md` | 三联表确定性算 flag；AI 只加定性、不碰数字、不预测业绩 |
| ⑤ 运营策略 | 已有上游诊断，要落成动作 | `references/运营策略.md` | 每条标来源(basis=哪份诊断)；禁业绩预测；缺上游列 gap |

走某一段时，**先把上表对应的文件读进来，再严格按它的纪律执行**。整链走就按 ⓪→⑤ 顺序，每段产一个简短摘要喂下一段。

> **横切查表**：主图/详情页/人群三段判"渠道适配·该主打什么·文案合不合规"时，查 `references/渠道与人群速查.md`（平台气质矩阵 + 购买决策驱动 + 平台合规禁用词），结论挂 basis。

## 📦 输出约定

- 每段结论分两类标清楚：`[数据]`＝有真实数据支撑；`[推断]`＝情境/逻辑推导，需真实数据回填。
- 整链走时，每段产一个简短摘要（结论 + cantJudge + 完备度），作下一段输入。
- 运营策略每条动作格式：`维度 · 优先级 · 动作 · basis(来源) · 时间窗`。
- 末尾统一附：本次**数据完备度 x/10** + **判断不了的清单**。

## ✅ 自检（产出前过一遍）

- [ ] 通篇无频率百分比（"X%的人会…"）、无业绩预测（"涨X%"）。
- [ ] 每条结论都有 `[数据]/[推断]/basis` 来源标注。
- [ ] 数据契约缺项已降级标注，没有硬给。
- [ ] 人群段：真的读了卡片核心段代入，不是名字印象；缺真实客群构成时只列障碍未排序。
- [ ] 整链：handoff 接上了（最致命障碍→详情页；客群错位→选盘；策略 basis 指回诊断）。
