# 电商运营诊断（总编排）

> 一台把「四诊断 ⟶ 运营策略」串成链的总台。你给一个产品 + 物料 + 真实数据，它按统一纪律带你走完整条诊断链，输出**可溯源、缺数据降级、不编不预测**的诊断与动作。
>
> 世界观：**真实数据至上**。模型擅长"这条障碍对这群人成不成立 / 这屏结构对不对 / 这个渠道是不是黑洞"（是非·分级·算术题），不擅长"百分之多少的人会怎样 / 改完涨多少"（频率·预测题）。本 Skill 只产前者，焊死不产后者。
>
> 📌 **出身**：本方法论经真实电商店铺整链实战检验、反复打磨——不是纸面框架。别的工具敢拍胸脯给你涨幅，本诊断只给**站得住的判断**。

## 🚦 触发

用户说「跑电商运营诊断」「给我这产品/店铺做个体检」「我这主图/详情页行不行」「谁会买」「这堆数据怎么读」「接下来该打什么」——进本 Skill，先判要走哪段（见路由表），再按对应方法论走。

## 🟥 贯穿红线（五段全程遵守，违任一返工）

1. **缺数据降级标注**——数据契约缺一不硬给结论，按降级路径走、并红标说明。
2. **不编百分比/频率**——「X% 的人会划走 / 命中率 30%」一律禁。频率题模型有效性≈0。
3. **不预测业绩**——CTR / 转化率 / 涨幅 / 销量 / GMV 任何数字预言一律禁；只给方向。
4. **来源必标**——每条结论挂 `[数据]` / `[推断]` / `basis`（哪几条情境/哪份诊断在起作用），可溯源。
5. **不自抓链接**——天猫/淘宝反爬抓不到；要诊断 → 发截图/文字。模型不替你抓、不替你编人口构成。
6. **不透支长期信任（儒家底线）**——短期 ROI 动作若伤害用户长期信任（虚假宣传、夸大功效、刷单刷评），一律标 `[信任透支]` 并禁止进入"推荐策略"。店铺的根基不是单次 GMV，是复购率和口碑复利。

> 产出前自己过一遍这六条，命中即改。

### 0. 维特根斯坦 · 术语审计（全段前置）

**核心问题**：「你说的『转化率』到底指什么？」

**铁律**：每次诊断开始前，先做术语对齐。不允许带着模糊定义就进诊断。

| 常见歧义词 | 须澄清的选择 |
|-----------|------------|
| 转化率 | 浏览→下单？下单→支付？浏览→支付？含退款？ |
| UV 价值 | 全店 UV？品类 UV？单品 UV？含广告？ |
| 客单价 | 成交均价？笔单价？含运费？ |
| 点击率 | 搜索曝光→点击？首页推荐→点击？ |

**输出格式**：诊断开始时附一行 `术语对齐：转化率=|...|UV价值=|...|` ，未对齐就标 `[待澄清]`。

> **对应贯穿红线④「来源必标」** — 连指标定义都没对齐，来源标注毫无意义。术语对齐是来源标注的前置条件。

### 1. 奥地利学派 · 需求动机还原

**核心问题**：「这个人搜这个词的时候，他心里缺什么？」

**来源**：米塞斯「人的行动是有目的的」。所有购买行为都是人在消除某种不安。

**怎么用**（激活于 ③人群验证）：
- 分析每个客群时，不只看标签（"25-35 岁女性"），追问：**她搜什么词进店、要解决什么场景下的什么焦虑**。
- 判断障碍是否成立的标准：**这道障碍是否真实阻挡了她消除那个不安？**
- 代入用户画像卡片时，不得跳过"她为什么来"这一步。

**输出线索**：人群段每条障碍加一行 `需求线：用户搜[关键词] ∧ 要解决[场景/不安]`。

**红线**：不得靠人群标签名称闭门猜需求。需求推断必须基于：用户搜索词 / 评论原话 / 竞品评价中重复出现的痛点词汇。需求来源未标 → 该障碍 basis 不可信。

#### 🔄 补充视角：存在主义 · 焦虑与障碍

> **互补关系**：奥地利学派回答「人为什么买」（需求动机），存在主义回答「人为什么不买」（存在焦虑 + 意义缺失）。

**核心问题**：「用户面对这个产品时，感受到什么存在焦虑？他怕踩什么坑？」

**来源**：萨特「存在先于本质」——人在行动前先感知到自己面对世界的无力感；海德格尔「此在」——人总是在特定处境中经验世界。

**怎么用**（激活于 ③人群验证，作为奥地利学派的反面补充）：
- 每分析一个客群的需求（她缺什么），**同时问**：她在购买过程中怕什么？犹豫什么？
- 障碍不只看功能性（"参数看不懂"），也看**存在性**（"买了没用怎么办" / "买错丢脸" / "别人会怎么看"）。

| 需求面（奥地利学派） | 焦虑面（存在主义） |
|---------------------|-------------------|
| 她想解决什么不安？ | 她怕买完之后产生什么新的不安？ |
| 搜索词反映什么渴望？ | 评论区差评反映什么恐惧？ |
| 主图怎么唤起渴望？ | 详情页怎么消除恐惧？ |

**输出线索**：人群段障碍加 `焦虑面：{用户怕什么} `（标 `[推断]`），与 `需求线：` 成对出现。

**红线**：焦虑推断同样不得闭门造车，必须基于评论中的担忧原话 / 问大家高频问题 / 退货原因。来源未标 → 不可信。

### 2. 第一性原理 · 指标原子化

**核心问题**：「这个指标的本质是什么？还能再拆吗？」

**来源**：亚里士多德→物理学→马斯克。不类比、不照搬行业惯例，从最底层要素重新推导。

**怎么用**（激活于 ④数据分析）：
- 拿到任何宏观指标，先拆到原子变量：

```
GMV
├── 访客数
│   ├── 曝光量
│   └── 点击率
│       ├── 主图吸引力
│       ├── 价格锚点
│       └── 标题匹配度
├── 转化率
│   ├── 加购率
│   ├── 下单率
│   └── 支付率
└── 客单价
    ├── 件单价
    └── 连带率
```

- 逐项排查，直到定位到**真正的原子级变量**出问题。
- **与系统论的联动**：拆到原子后，用系统论判断"改哪个原子对整链影响最大"。

**红线**：不拿"转化率跌了"当结论——那只是起点。必须追到"转化率跌是因为加购率跌，加购率跌是因为新详情页第三屏跳失率翻倍"这个粒度。

### 3. 系统论 · 约束定位

**核心问题**：「整条链的瓶颈在哪？不是哪个环节最差，是哪个环节卡住了后面的一切。」

**来源**：贝塔朗菲一般系统论 + 高德拉特约束理论（TOC）。

**怎么用**（激活于 ②详情页、④数据分析）：

**三条规则**：

1. **约束唯一律**（TOC）：任何时刻整条转化链只有一个最紧约束。改非约束环节的投入，整链产出不增加。

2. **次优陷阱**：最致命障碍是详情页结构，你却优化了客服话术 → 完全无效。

3. **非线性和延迟**：改了一个环节的效果不会立刻显现。别在 3 天内下结论——流量需要时间沉淀，转化数据有滞后。

**判断约束的方法**：
- 流量漏斗中，**最后一个"正常→陡降"的拐点** = 当前约束。
- 漏斗整体平缓但绝对值低 → 约束在漏斗入口（曝光/点击）。
- 漏斗入口高但尾部低 → 约束在转化环节。

**输出格式**：诊断结论中标 `系统约束：|{环节}| 证据：{数据}`。改完约束后，下一轮诊断重新定位新约束。

#### 🔄 补充视角：辩证法 · 矛盾转化

> **互补关系**：系统论/TOC 告诉你「现在瓶颈在哪」，辩证法告诉你「瓶颈会自己挪」。

**核心问题**：「上个月不是瓶颈的环节，这个月是不是升级了？」

**来源**：毛泽东《矛盾论》——主要矛盾和次要矛盾在特定条件下会互相转化；矛盾的主次方面也会转化。

**怎么用**（激活于 🔄复诊 + 🔁自进化）：

1. **矛盾转移追踪**：每次复诊时，不仅判上一次的约束消解没有，也检视各段上次的"非瓶颈"是否有**升格迹象**。
   - 上次标"次要"的障碍→这次数据是否变了→是否已升级为主要约束。
   - 上次"可控域"的变化是否制造了新的主要矛盾。

2. **转化信号**：
   - 某次要障碍的关联指标连续两次诊断都在恶化 → 标 `⚠️ 矛盾转化预警`。
   - 竞品动作引发的新约束 → 原来的策略可能从"正确"变为"过时"。

3. **动态视角规则**：每次诊断不得默认"上次的约束格局仍成立"。必须先质疑：「距离上次诊断，局面发生了什么变化，矛盾的主次有没有重新洗牌？」

**输出格式**：复诊时加一行 `矛盾动态：上次次要→本次升级=|...| / 上次约束仍成立?=是/否`。

### 4. 博弈论 · 竞品反应预判

**核心问题**：「我动了，对手会怎么回应？」

**来源**：纳什均衡、囚徒困境、序贯博弈。

**怎么用**（激活于 ⑤运营策略）：

每条策略动作，额外加一个 **竞品反应预判** 字段：

| 动作类型 | 竞品最可能的回应 | 你需要准备什么 |
|---------|----------------|--------------|
| 降价 | 跟降（价格战） | 降本预案 / 差异化壁垒 |
| 改主图风格 | 抄袭（1-2 周后跟进） | 持续迭代节奏 |
| 换关键词策略 | 高价抢排名 | 长尾词矩阵防守 |
| 推新品 | 观望→降价狙击 | 窗口期测算 |

**输出格式**：运营策略段每条动作加 `竞品回应：{预判} · 博弈建议：{对策}`。

**红线**：博弈论只用于判断对手的可行回应（是非题：他能不能跟？），不用于预测"他会降 10% 还是 20%"（频率题）。

### 5. 贝叶斯 · 先验→后验回路

**核心问题**：「我在看到新证据之前信什么？新证据出来后我该怎么更新？」

**来源**：贝叶斯定理 — 后验概率 ∝ 似然 × 先验概率。

**怎么用**（嫁接于 🔁自进化回路 + 🔄复诊）：

```
先验（Prior）→ 诊断前读经验库已有判例
    │
似然（Likelihood）→ 本次诊断观察（主图/详情/人群/数据段产出）
    │
后验（Posterior）→ 更新后的判断 → 写回经验库 §0
    │
证据（Evidence）→ 只有真实成交数据回填才能升级置信度
```

**铁律**：
- **不得跳过先验**：诊断前必须读经验库，等于必须先有 prior。裸诊断 = 无先验的后验 = 无效。
- **不得自证**：后验不能由模型自己验证。只有真实成交数据回填才能把后验升级为新的先验。这是 🔁 回路的哲学基础。
- **置信度演化**：假说(§0) → 单验(§1) → 规律(§2) = 先验越来越强。对应贝叶斯的"后验做下一次的先验"。

**与现有自进化回路的对应**：
- 意义闸（命题7）= 贝叶斯框架下的「这个假说能不能被证据检验？」
- §0 待验证隔离区 = 后验等待证据
- 棘轮升降级 = P(H|E) 更新
- 禁止模型自评 = 「证据只能来自现实，不能来自模型自己」

### 6. 斯多葛 · 控制圈边界

**核心问题**：「这堆问题里，哪些是我能控制的？哪些只能观察？」

**来源**：爱比克泰德「有些事在我们掌控之中，有些事不在」。

**怎么用**（激活于 ⑤运营策略收尾 + 💬复诊收尾）：

每条策略/诊断收尾，强制标注控制圈：

| 可控（行动域） | 不可控（观察域） |
|---------------|----------------|
| 主图/详情页/视觉 | 平台算法规则变化 |
| 投放预算分配 | 竞品定价策略 |
| 客服话术/响应速度 | 大环境消费力 |
| 评论维护节奏 | 竞品投放力度 |
| 活动报名/节奏 | 大盘流量波动 |

**铁律**：
- **可控的 → 立即给行动清单**（优先级排序 + 时间窗）。
- **不可控的 → 只设观察哨**（"如果 X 变化超过阈值 Y，则触发 Z 预案"），不输出焦虑型判断（如"平台流量下降你完了"——你控不了的事不值得焦虑）。
- **灰色地带 → 拆**（"DSR 评分" = 客服话术你控 + 物流速度你半控 + 恶意差评你不可控，拆开分别处理）。

> **对应贯穿红线③「不预测业绩」** — 预测本质上是在不可控域里做判断。斯多葛帮你区分：可控域里的判断叫「策略」，不可控域里的判断叫「算命」。

## 📋 数据契约（开工门槛）

| 段 | 必填 | 缺了怎么办 |
|----|------|-----------|
| 通用（每段都要） | 品类 / 客单价 / 人群 / 参数 / 竞品（+渠道） | 对应维度判断标 `[推断]`、basis 留空并注明"情境缺失"，不硬给 |
| 人群验证 额外 | 真实客群构成（人群占比 + 来源） | **降级**：只列障碍、全程"排序不可信"红标，不排序 |
| 数据分析 | 流量结构表 / 客群错位表 / SKU 矩阵表（手填真实数） | 哪张表空，该维度不算，不编数字 |

**数据完备度评分**：每段输出末尾附「数据完备度 x/10」。满分 10，每缺一个通用维度 −1.5，无视觉能力 −2。

## 🔗 链路与 handoff

```
⓪真实数据采集(评论/后台 → 客群占比·痛点·三联表)
        │ 喂真实输入(红线⑤要的真实客群构成就来自这)
        ▼
主图诊断 ─→ 详情页诊断 ─→ 人群验证 ──最致命障碍──▶ 详情页"怎么改"(手术)
                            ▲
            数据分析 ──客群错位──┘ (指导该代入哪些有份量的人群)
                │
            三份摘要(详情页 + 人群 + 数据) ──▶ 运营策略(按优先级出动作)
```

- **整链走**：主图→详情页→人群→数据→策略，前段产物喂后段。
- 关键接力：
  - 人群验证产「最致命障碍」→ 喂详情页"怎么改"，省掉"拍人群"翻车。
  - 数据分析识别「客群错位」（访问端 vs 支付端不一致）→ 指导人群验证该代入谁。
  - 运营策略自动消费 ①②③ 三份最新诊断摘要；缺哪份标"未做"、不基于它下结论。

| ② 详情页诊断 | 审整套详情页 / 详情页怎么改 | `子技能/详情页架构师/SKILL.md` | 🧠系统论·约束定位 — 审五维(结构/排序/视觉/因果/转化)；basis 取6维；禁涨幅预测 |
| ③ 人群验证 | 想知道"谁会卡、哪条障碍最致命" | `子技能/人群验证引擎/SKILL.md` | 🧠奥地利学派·需求还原 — 8红线；严重度四档；每障碍带"❗判断不了的"+需求线；禁频率%；**必实际读卡片代入** |

走某一段时，**先把上表对应的文件读进来，再严格按它的纪律执行**。整链走就按 ⓪→⑤ 顺序，每段产一个简短摘要喂下一段。

> **横切查表**：主图/详情页/人群三段判"渠道适配·该主打什么·文案合不合规"时，查 `references/渠道与人群速查.md`（平台气质矩阵 + 购买决策驱动 + 平台合规禁用词），结论挂 basis。

## 🔁 自进化 · 判例制回路（🧠贝叶斯·先验→后验）

> 哲学：**让现实当唯一的老师，模型永远不许给自己打分。** 这台 skill 变准，不是模型"学会"了什么，而是**现实的判例攒多了、被归档了**。经验库 ＝ 判例卷宗，不是心得笔记。详见 `data/经验库.md`。
>
> 这也是把红线焊死的东西今天禁止的答案（"某人群某款转化高"）合法赚回来的唯一路径：它不是模型猜的，是攒够真实成交后从 `[推断]` 升 `[数据]`。
>
> 💡 **贝叶斯视角**：经验库 = 先验概率，每次新诊断 = 似然函数，真实成交回填 = 证据。P(H|E) ∝ P(E|H) × P(H)。详见 🧠认知操作系统 §5。

**三个动作，每次诊断都走：**

1. **启动·读先验**——开诊断前先 Read `data/经验库.md` 与本次相关的段/品类章节，把判例当先验，引用标"据经验库 §X（置信态/证据）"。
2. **收尾·回写假说（硬性）**——每次诊断结束**必做**：把本次新产生的是非假说写进经验库 §0 待验证隔离区（标来源/日期/证伪条件）。这是流程，不是美德；漏了＝本次没收尾。
3. **回填·棘轮升降级**——用户带真实结果回来时才动：假说被真实数据证实 → 升「单验/规律」；被推翻 → 退役到 §4。**升级入口只有这一个，模型自评不算。**

**回路红线（违任一不写库）：**
- **意义闸（写库第一道·命题7）**：写任何条目前先过三问——①是不是是非命题（频率/占比＝伪命题，直接沉默、连 §0 都不进）？②是不是冒充知识的预测（只能当"待裁决问题"记 §0，永不作结论）？③说得出证伪条件吗（说不出＝废话，拒收）？不过闸连假说都不收。
- 只有**真实结果**能升置信；模型印象只能待在 §0，禁止当 basis。
- **频率/预测永不入库**——能沉淀的只有是非命题（障碍成立与否、渠道黑洞、王者 SKU），不许沉淀百分比或涨幅。
- **引用纪律**：只有「单验」及以上能进 basis；§0 的假说绝不当结论。
- **时钟是"真实数据到达"，不是时间**：没有新真实结果就别重跑评估（重跑只会把同一批结论再嚼一遍）。要催回填→挂 `/schedule` 提醒，loop 只催人喂真数据、不替人造数据。

## 🔄 复诊 · 收敛循环（🧠斯多葛·控制圈 + 贝叶斯·证据驱动）

> 整链不是一次性流水线，是会**收敛**的环。诊断 → 你去改 → 带着"改了什么"回来 → **只复诊 delta** → 判最致命障碍消解没 → 没消解再来一轮。
>
> 和自进化（🔁，跨会话）是两层、别混：**复诊判"结构问题改没改"**（是非题，模型能判）；**自进化判"改了之后真实数字动没动"**（回填，只有现实能判）。复诊的结论是后者的假说来源。

**何时进复诊**（必须有真实 delta，否则不跑——没有新改动/新数据的"复诊"＝原地打转、镜厅空转）：
- 用户说"按你说的改了，再看看"，或给了改后的新物料 / 新真实数据。

**复诊四步：**
1. **取上次结论**：读上次诊断摘要（最致命障碍 / 各段红 flag / 完备度）。没存档就退化成首诊。
2. **认 delta**：这次改了什么、多了哪些真实数据。**只诊断变化面，不整链重跑。**
3. **判消解**（逐条对上次的红 flag）：
   - ✅ **消解**：结构问题真补上了（是非题，模型判）。
   - 🔁 **搬家**：旧障碍没了、改动又引出新障碍 → 记下，继续。
   - ❌ **仍在**：没改到点上 → 说清差在哪。
4. **判收敛 or 再迭代。**

**退出条件（收敛即停，别空转）：**
- 最致命障碍**消解** 且 无新红 flag → **收敛，停**。
- 完备度不再提升、障碍只在原地搬家 → **停**，说明"已到模型能判的边界，再循环无新信息"（命题5.6：到边界了别往不可说那边推）。
- **最多 3 轮**护栏：到顶仍未收敛 → 停、列"还卡在哪 + 需要什么真实数据/动作才能破"，交回用户，不自我空转。

> 💡 **斯多葛视角**：复诊收敛后的收尾，强制标注本轮行动的控制圈——哪些障碍是你改了之后解决的（可控），哪些受制于外部因素未解决（不可控→设观察哨）。不把不可控的失败当自己的失败。

**复诊收尾**：照常走自进化收尾——把"某障碍改 X 后是否消解"作为新假说写进经验库 §0（过意义闸、标证伪条件），等真实数字回填时再升降级。

## 📦 输出约定

- **前置**：诊断开始附 `术语对齐：转化率=|...|UV价值=|...|`（🧠维特根斯坦）。
- 每段结论分两类标清楚：`[数据]`＝有真实数据支撑；`[推断]`＝情境/逻辑推导，需真实数据回填。
- 整链走时，每段产一个简短摘要（结论 + cantJudge + 完备度），作下一段输入。
- ④数据分析：附原子变量拆解树，标 `根因：|{原子变量}|`（🧠第一性原理）。
- ⑤运营策略每条动作格式：`维度 · 优先级 · 动作 · basis(来源) · 竞品回应 · 控制圈 · 时间窗`（🧠博弈论 + 斯多葛）。
- 末尾统一附：本次**数据完备度 x/10** + **判断不了的清单** + **控制圈标注**（可控动作 / 不可控观察哨）。

## ✅ 自检（产出前过一遍）

- [ ] 通篇无频率百分比（"X%的人会…"）、无业绩预测（"涨X%"）。
- [ ] 每条结论都有 `[数据]/[推断]/basis` 来源标注。
- [ ] 数据契约缺项已降级标注，没有硬给。
- [ ] 人群段：真的读了卡片核心段代入，不是名字印象；缺真实客群构成时只列障碍未排序。
- [ ] 整链：handoff 接上了（最致命障碍→详情页；客群错位→选盘；策略 basis 指回诊断）。
- [ ] 自进化：开诊断前读了经验库相关章（🧠贝叶斯·先验）；收尾把本次假说回写进 §0（过意义闸、标证伪条件）；引用的 basis 只取「单验」及以上。
- [ ] 复诊（如适用）：有真实 delta 才跑；逐条判了上次红 flag 消解/搬家/仍在；收敛或到 3 轮护栏就停，复诊收尾标了控制圈（🧠斯多葛）。
- [ ] 🧠 认知OS：术语已对齐（维特根斯坦）；数据分析有原子变量拆解（第一性原理）；策略有竞品回应预判（博弈论）；收尾有控制圈标注（斯多葛）。
- [ ] 🧠 补充：人群段需求线+焦虑面成对出现（存在主义）；复诊有矛盾动态标注（辩证法）；策略无害信任透支动作（儒家红线⑥）。
