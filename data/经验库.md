# 电商运营诊断 · 经验库（判例制）

> 运行口径：分发版 v1.8.0 **空模板**——判例只能从你自己的真实诊断中积累，别拿别人的判例当先验。单店状态不进本库——写 `data/诊断存档/`（两库分工见 SKILL.md 🗂️）。
>
> **哲学一句话**：让现实当唯一的老师，模型永远不许给自己打分。
>
> 本库不是模型的"心得笔记"，是**现实判例的卷宗**。这台 skill 变准，不是因为模型学会了什么，而是因为**现实的证词攒多了、被归档了**。
>
> 模型在这里的角色被焊死：**只当法官（判是非/分级）和算盘（算术），从不当证人（提供事实）和先知（预测）**。证人席只有真实数据能坐，先知席永远空着。

---

## 🟥 进化红线（写库前自检，违任一不写）

1. **只有真实结果能升置信**——模型自评 / 印象 / "这次诊断挺准"一律不进可信层，只能进 §0 待验证隔离区。
2. **频率 / 预测永不入库**——再多样本也不准沉淀"X% 的人会…""加这屏 +Y% 转化"。能入库的只有**是非命题**：障碍 X 对人群 Y 成立 / 不成立、渠道 Z 是黑洞、SKU W 是王者。
3. **单次不成规律**——一次诊断只产**假说**（进 §0），不能直接当经验复用。
4. **每条判例可溯源**——必带 `主张 · 置信态 · 来源案例 · 日期 · 证据 · 适用边界`，随时能回滚。
5. **会证伪、会退役**——被新真实数据推翻的旧判例必须降级 / 退役到 §4，不许闷着留。只进不退 = 教条堆积。

---

## 🪜 置信阶梯（每条判例只能靠真实证据往上爬一格）

```
假说(H)            一次诊断·模型推断      → 只能待在 §0 隔离区，禁止当 basis 引用
  │ ↑ 真实结果证实
单验(V1)           1 份真实结果证实它成立   → 进各段判例(§1)，可当 basis，但标"单案例"
  │ ↑ 再被证实（N≥2 次，或跨品类/跨店复现）
规律(R)            反复/跨案例复现          → 晋升 §2 跨段规律，可放心当 basis
  │ ↓ 被新真实数据推翻（任何阶段都可能）
退役(X)            证伪                    → 进 §4，划掉但留档，注明被哪份数据推翻
```

**引用纪律**：诊断时只有 **单验(V1) 及以上**能写进 basis，引用须注明"据经验库 §X（置信态/证据）"。**假说(H) 绝不当结论用**——它只是下次该向用户回填验证的问题。

## 🧱 判例的一般形式（形式只此一种 · 命题6）

> 维特根斯坦：所有命题共享同一个一般形式，**形式是一、事实是多**。本库的判例也只此一种形式——**全链唯一**。各叶子库（详情页/主图/人群…）只**实例化并引用**本形式，绝不各自重写一套治理（形式说一次、事实显示多处，否则就是漂移）。

一条判例 ＝ ⟨**主张**（是非命题）· **证伪条件**（什么真实结果会让它为假）· **证据**（哪份真实数据）· **来源案例** · **日期** · **置信态** · **适用边界**⟩

**库怎么分（命题6 推论）**：治理（意义闸/置信阶梯/红线/一般形式）＝全链唯一，写本文件这一处；具体判例按**事实归属**分布——主图/详情页/人群各自库（随产品走）、数据/策略/**跨段**判例归本库。叶子库只"遵循本库判例制 ＋ 记自己的事实"，不复制治理。分发时才把治理 inline 进各产品包。

## 🚪 意义闸（写库第一道 · 不过此关，连假说都不收 · 命题7）

> 命题7：凡不可说的，必须沉默。下面三问，任一不过 → **不写库**。

1. **是不是是非命题？** 频率/占比（"X% 的人会…"）＝伪命题（unsinnig），假装描画事实却什么都没画 → **直接沉默，连 §0 都不进**。
2. **是不是冒充知识的预测？** "改完涨 X%" 是假说、不是知识（命题6.36311：太阳明天升起也只是假说）→ 只能当"待现实裁决的问题"记进 §0，**永不作结论**。
3. **说得出证伪条件吗？** 答不出"什么真实结果会让它为假"＝套套逻辑/废话（双极性失败）→ **拒收**。

> 意义闸判"有没有意义"，置信阶梯判"有没有被证实"——**先过意义闸，再上置信阶梯**。

## 维护规则（回路怎么转）

- **启动动作**：开诊断前，先 Read 本库与本次相关的段/品类章节，把判例当先验，引用标"据经验库 §X"。同店在 §0 主表有未回填假说 → 开场先催（沉睡区不催）；催的入口是该店最新诊断存档 frontmatter 的「未回填假说」指针。
- **收尾动作（硬性）**：每次诊断结束**必做**——把本次新产生的是非假说写进 §0（标来源/日期/复诊计划）。这是流程，不是美德。
- **回填棘轮（唯一的升级入口）**：用户带真实结果回来时——
  - 假说被证实 → §0 移入 §1，置信态升 V1；再被复现 → 升 §2 规律。
  - 假说/旧判例被证伪 → 移入 §4 退役，注明被哪份数据推翻。
- **时钟是"真实数据到达"，不是时间**：没有新真实结果就不要重跑评估（重跑只会把同一批结论再嚼一遍）。需要催回填时挂 `/schedule` 提醒，loop 的活是**催人喂真数据，不替人造数据**。
- **写入方式**：Edit 追加到对应章节，不 Write 覆盖全文。
- **冲突处理**：新旧判例冲突 → 标日期保留两条，让用户判，别自行抹掉。
- **定期清理**：§0 主表条目超 14 天未回填 → 移入 §0·附沉睡区（不删不催、真实数据到达即唤醒回主表）；§4 不清理（账本只增）。
- **超 500 行后**：按章节读，不全文读。
