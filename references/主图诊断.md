# 主图诊断（电商运营诊断 · 第①段 · 独立版）

> 看单张主图在「这个情境」下行不行、该怎么改。**无普世"好图"标准**，好坏全由情境(品类×客单价×人群×竞品×渠道)推导。
> 独立版：你直接发主图 + 情境，我（有视觉的模型）自己看图做三层判断，**不依赖任何网页工具 / OCR 引擎**。
> 口径源自方法论「主图情境诊断」的三层判断。

---

## 触发

用户发一张主图 + 情境，问"这图行不行 / 怎么改"。**发图，别发链接**——天猫/淘宝反爬，链接抓不到只会编。

## 输入

- 主图（直接贴图）
- 情境：品类 / 客单价 / 人群 / 竞品 / 渠道（+参数）

---

## 三层判断（逐层来，不跳层）

**第①层 · 平台硬规则（能查的硬标准，与情境无关）**
- 主体占比够不够（主图主体一般应 ≥ 画面 50%）。
- 有没有大面积"牛皮癣"促销贴片超出平台容忍；有没有违禁/夸大词。
- 文字是否过多挡住主体。
- → 命中这层=必改项，跟情境无关。

**第②层 · 客观测量（看图估，标"目测"）**
- 文字占比（高/中/低）、信息密度、主体清晰度。无 OCR 就肉眼估、标明是目测。

**第③层 · 情境判断（核心，从情境推导）**
- **品类识别度**：3 秒内看不看得出这是什么品类。
- **差异卖点**：竞品都在喊的点 vs 这张图喊的点，有没有"人无我有 / 人有我优"。
- **人群调性**：画面气质对不对目标人群（客单价 × 人群一起看）。
- **文案因果**：主图文案是"产品属性 + 解决方案"的因果句，还是空洞形容词。
- **渠道适配**：货架页(搜索/推荐)首图要素 ≠ 详情页首图要素，按渠道判。

**第④层 定性视觉 + 第⑤层 改进**
- 定性看图项（识别度/差异卖点/人群调性/牛皮癣）各给 green/yellow/red + 一句 note。
- 改进按优先级给 2-5 条，每条挂 `basis`（哪几条情境在起作用）、可直接执行。

---

## 铁律（与全链红线一致）

- 每条好坏必须从情境推导，`basis` 写明哪几条在起作用。
- 缺对应情境 → 标 `[推断]` / 待补，**不硬给结论**。
- **严禁任何点击率 / 转化率 / 涨幅百分比预测**，改进只给方向。

## 输出

三层判断结论 + 定性视觉(挂 level) + 改进清单(挂 basis/优先级) + 「数据完备度 x/10」+ 「判断不了的」（需哪些真实数据才能从 [推断] 升 [数据]）。

## 往下接

主图诊断完 → 进**详情页诊断**（看整页）；若想知道"这张图会卡住谁" → 把图喂**人群验证**。
