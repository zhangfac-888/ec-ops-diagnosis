# 真实数据采集（电商运营诊断 · 第⓪段 · 真实输入入口）

> 全链的地基。本诊断**红线就是"没真数据不开工"**——但真数据从哪拿、怎么洗、产出什么喂给后面，这一段说清楚。
> 把"零散评论 / 后台报表"变成后面各段能直接吃的**真实输入**：客群占比、真实痛点、参照物、流量/客群/SKU 三联表。
> 核心纪律不变：**真实计数可以用（数评论、数订单），样本不代表盘子时必须标'仅此样本'，绝不替你编。**

---

## 这一段产什么、喂给谁

| 我帮你产出 | 喂给哪段 | 对应红线 |
|----------|---------|---------|
| 真实客群占比（谁在买，带来源） | ③ 人群验证（红线⑤要的真实客群构成） | 缺它人群段只能降级不排序 |
| 真实痛点 / 高频负面根因 | ② 详情页诊断（该补什么信任状） | 来源必标 |
| 参照物（用户拿谁跟你比） | ② 详情页 + 竞品维度（差异化从哪切） | — |
| 流量/客群/SKU 三联表 | ④ 数据分析 | 手填真实数，不编 |

## 取数速查（评论 + 经营数据，按平台）

| 平台 | 评论怎么导 | 经营/客群数据怎么导 | 坑 |
|---|---|---|---|
| 淘宝/天猫 | 商家后台「评价管理」导出；无权限用采集插件 | **生意参谋**（流量来源/客群分析/竞店分析）；店透视 | 默认好评极多，体检要狠；追评单独成列 |
| 京东 | 商智后台导出；商品页采集 | 京东商智 | "晒图评论"质量高；"问大家"是金矿 |
| 拼多多 | 商家后台「评价」导出 | 多多情报通 | 评价偏短，大量"还行"无效，体检要狠 |
| 抖音/抖店 | 抖店后台「商品评价」 | 抖店罗盘/巨量 | 视频评论区 ≠ 商品评价，别混 |
| 小红书 | 笔记评论需采集（无官方导出） | 蒲公英后台 | 多为"种草/求链接"，按"是否真实使用过"过滤 |
| 亚马逊 | 卖家后台 Review 报告；Helium10/JS 导出 | 品牌分析(ABA) | 英文为主，保留英文关键词+中文释义 |
| 独立站 | 评论 App(Judge.me/Loox/Yotpo)导 CSV | GA4 / Shopify Analytics | Loox 带图评论质量高；可关联订单看复购 |

> SKU 维度的真实数，可用「淘宝SKU批量导出」类工具批量导出，直接填进 ④ 的 SKU 矩阵表。

## 两条取数轨

**轨 A：评论 → 客群 / 痛点 / 参照物**（至少 100 条，每行一条，含正文+评分+时间+规格）
**轨 B：后台 → 流量 / 客群占比 / SKU**（生意参谋/店透视等，导出原始数）

---

## 📋 可直接粘贴的提示词（任意 AI 都能跑：豆包/Kimi/ChatGPT/Claude）

**① 数据体检（上传评论文件后发）**
```
先别分析。请对我上传的评论做数据质量检查并输出《数据质量检查表》：
1) 原始行数 vs 有效评论数；2) 字段完整性；3) 识别空评/默认好评；
4) 是否混入别的SKU/商品；5) 时间与规格是否混在一列要拆；6) 建议的有效样本口径。
先给检查结果+排除规则，等我确认再处理。
```

**② 客群 + 痛点 + 参照物提炼（确认数据干净后）**
```
基于干净评论，输出三块（全部只用真实评论、不过度推断、低频别放大、每块附代表原话）：
A 购买人群占比（用真实计数，标"占本样本X%"，并提醒此样本是否代表全店）；
B 高频负面根因，每条判为：产品问题 / 预期管理问题(详情页吹过头) / 沟通问题(客服没讲清)；
C 参照物：用户拿谁比（含竞品牌 / "别家" / 本产品旧版 / 同店其他SKU），各附原话和倾向。
```

> 产出的「客群占比」喂 ③ 人群验证时，务必带上"来源=评论样本"和"是否代表全店"的判断——这正是红线⑤要的真实构成，不是模型编的。

---

## 体检要剔除/拆分的脏数据

默认好评 / "未填写评价" · 重复刷评（虚高频次） · 混入的别的 SKU · 时间和规格混一列 · 模板化刷单评。
