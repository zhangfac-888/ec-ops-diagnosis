# 认知操作系统 · 七维思维范式（电商运营诊断 · 横切层）

> 本文件是从 SKILL.md 拆出的完整方法论正文（v1.7.1 起按需读取）。主文件只留速查路由表；**深入任何一段前，先读本文件对应章节再产出。**

> 五段诊断是「做什么」，七维是「怎么思考」。每段诊断默认激活一个认知范式，让你的判断不只在方法论层面对，更在思维底层有根。v1.7.0 新增三向补充（存在主义·焦虑障碍 / 辩证法·矛盾转化 / 儒家·信任底线），分别嫁接在奥地利学派、系统论、贯穿红线之下，不另立维度。

### 0. 维特根斯坦 · 术语审计（全段前置）

**核心问题**：「你说的『转化率』到底指什么？」

**铁律**：每次诊断开始前，先做术语对齐。不允许带着模糊定义就进诊断。

| 常见歧义词 | 须澄清的选择 |
|-----------|------------|
| 转化率 | 浏览→下单？下单→支付？浏览→支付？含退款？ |
| UV 价值 | 全店 UV？品类 UV？单品 UV？含广告？ |
| 客单价 | 成交均价？笔单价？含运费？ |
| 点击率 | 搜索曝光→点击？首页推荐→点击？ |

**输出格式**：诊断开始时附一行 `术语对齐：转化率=|...|UV价值=|...|` ，未对齐就标 `[待澄清]`。

> **对应贯穿红线④「来源必标」** — 连指标定义都没对齐，来源标注毫无意义。术语对齐是来源标注的前置条件。

### 1. 奥地利学派 · 需求动机还原

**核心问题**：「这个人搜这个词的时候，他心里缺什么？」

**来源**：米塞斯「人的行动是有目的的」。所有购买行为都是人在消除某种不安。

**怎么用**（激活于 ③人群验证）：
- 分析每个客群时，不只看标签（"25-35 岁女性"），追问：**她搜什么词进店、要解决什么场景下的什么焦虑**。
- 判断障碍是否成立的标准：**这道障碍是否真实阻挡了她消除那个不安？**
- 代入用户画像卡片时，不得跳过"她为什么来"这一步。

**输出线索**：人群段每条障碍加一行 `需求线：用户搜[关键词] ∧ 要解决[场景/不安]`。

**红线**：不得靠人群标签名称闭门猜需求。需求推断必须基于：用户搜索词 / 评论原话 / 竞品评价中重复出现的痛点词汇。需求来源未标 → 该障碍 basis 不可信。

#### 🔄 补充视角：存在主义 · 焦虑与障碍

> **互补关系**：奥地利学派回答「人为什么买」（需求动机），存在主义回答「人为什么不买」（存在焦虑 + 意义缺失）。

**核心问题**：「用户面对这个产品时，感受到什么存在焦虑？他怕踩什么坑？」

**来源**：萨特「存在先于本质」——人在行动前先感知到自己面对世界的无力感；海德格尔「此在」——人总是在特定处境中经验世界。

**怎么用**（激活于 ③人群验证，作为奥地利学派的反面补充）：
- 每分析一个客群的需求（她缺什么），**同时问**：她在购买过程中怕什么？犹豫什么？
- 障碍不只看功能性（"参数看不懂"），也看**存在性**（"买了没用怎么办" / "买错丢脸" / "别人会怎么看"）。

| 需求面（奥地利学派） | 焦虑面（存在主义） |
|---------------------|-------------------|
| 她想解决什么不安？ | 她怕买完之后产生什么新的不安？ |
| 搜索词反映什么渴望？ | 评论区差评反映什么恐惧？ |
| 主图怎么唤起渴望？ | 详情页怎么消除恐惧？ |

**输出线索**：人群段障碍加 `焦虑面：{用户怕什么} `（标 `[推断]`），与 `需求线：` 成对出现。

**红线**：焦虑推断同样不得闭门造车，必须基于评论中的担忧原话 / 问大家高频问题 / 退货原因。来源未标 → 不可信。

### 2. 第一性原理 · 指标原子化

**核心问题**：「这个指标的本质是什么？还能再拆吗？」

**来源**：亚里士多德→物理学→马斯克。不类比、不照搬行业惯例，从最底层要素重新推导。

**怎么用**（激活于 ④数据分析）：
- 拿到任何宏观指标，先拆到原子变量：

```
GMV
├── 访客数
│   ├── 曝光量
│   └── 点击率
│       ├── 主图吸引力
│       ├── 价格锚点
│       └── 标题匹配度
├── 转化率
│   ├── 加购率
│   ├── 下单率
│   └── 支付率
└── 客单价
    ├── 件单价
    └── 连带率
```

- 逐项排查，直到定位到**真正的原子级变量**出问题。
- **与系统论的联动**：拆到原子后，用系统论判断"改哪个原子对整链影响最大"。

**红线**：不拿"转化率跌了"当结论——那只是起点。必须追到"转化率跌是因为加购率跌，加购率跌是因为新详情页第三屏跳失率翻倍"这个粒度。

### 3. 系统论 · 约束定位

**核心问题**：「整条链的瓶颈在哪？不是哪个环节最差，是哪个环节卡住了后面的一切。」

**来源**：贝塔朗菲一般系统论 + 高德拉特约束理论（TOC）。

**怎么用**（激活于 ②详情页、④数据分析）：

**三条规则**：

1. **约束唯一律**（TOC）：任何时刻整条转化链只有一个最紧约束。改非约束环节的投入，整链产出不增加。

2. **次优陷阱**：最致命障碍是详情页结构，你却优化了客服话术 → 完全无效。

3. **非线性和延迟**：改了一个环节的效果不会立刻显现。别在 3 天内下结论——流量需要时间沉淀，转化数据有滞后。

**判断约束的方法**：
- 流量漏斗中，**最后一个"正常→陡降"的拐点** = 当前约束。
- 漏斗整体平缓但绝对值低 → 约束在漏斗入口（曝光/点击）。
- 漏斗入口高但尾部低 → 约束在转化环节。

**输出格式**：诊断结论中标 `系统约束：|{环节}| 证据：{数据}`。改完约束后，下一轮诊断重新定位新约束。

#### 🔄 补充视角：辩证法 · 矛盾转化

> **互补关系**：系统论/TOC 告诉你「现在瓶颈在哪」，辩证法告诉你「瓶颈会自己挪」。

**核心问题**：「上个月不是瓶颈的环节，这个月是不是升级了？」

**来源**：毛泽东《矛盾论》——主要矛盾和次要矛盾在特定条件下会互相转化；矛盾的主次方面也会转化。

**怎么用**（激活于 🔄复诊 + 🔁自进化）：

1. **矛盾转移追踪**：每次复诊时，不仅判上一次的约束消解没有，也检视各段上次的"非瓶颈"是否有**升格迹象**。
   - 上次标"次要"的障碍→这次数据是否变了→是否已升级为主要约束。
   - 上次"可控域"的变化是否制造了新的主要矛盾。

2. **转化信号**：
   - 某次要障碍的关联指标连续两次诊断都在恶化 → 标 `⚠️ 矛盾转化预警`。
   - 竞品动作引发的新约束 → 原来的策略可能从"正确"变为"过时"。

3. **动态视角规则**：每次诊断不得默认"上次的约束格局仍成立"。必须先质疑：「距离上次诊断，局面发生了什么变化，矛盾的主次有没有重新洗牌？」

**输出格式**：复诊时加一行 `矛盾动态：上次次要→本次升级=|...| / 上次约束仍成立?=是/否`。

### 4. 博弈论 · 竞品反应预判

**核心问题**：「我动了，对手会怎么回应？」

**来源**：纳什均衡、囚徒困境、序贯博弈。

**怎么用**（激活于 ⑤运营策略）：

每条策略动作，额外加一个 **竞品反应预判** 字段：

| 动作类型 | 竞品最可能的回应 | 你需要准备什么 |
|---------|----------------|--------------|
| 降价 | 跟降（价格战） | 降本预案 / 差异化壁垒 |
| 改主图风格 | 抄袭（1-2 周后跟进） | 持续迭代节奏 |
| 换关键词策略 | 高价抢排名 | 长尾词矩阵防守 |
| 推新品 | 观望→降价狙击 | 窗口期测算 |

**输出格式**：运营策略段每条动作加 `竞品回应：{预判} · 博弈建议：{对策}`。

**红线**：博弈论只用于判断对手的可行回应（是非题：他能不能跟？），不用于预测"他会降 10% 还是 20%"（频率题）。

### 5. 贝叶斯 · 先验→后验回路

**核心问题**：「我在看到新证据之前信什么？新证据出来后我该怎么更新？」

**来源**：贝叶斯定理 — 后验概率 ∝ 似然 × 先验概率。

**怎么用**（嫁接于 🔁自进化回路 + 🔄复诊）：

```
先验（Prior）→ 诊断前读经验库已有判例
    │
似然（Likelihood）→ 本次诊断观察（主图/详情/人群/数据段产出）
    │
后验（Posterior）→ 更新后的判断 → 写回经验库 §0
    │
证据（Evidence）→ 只有真实成交数据回填才能升级置信度
```

**铁律**：
- **不得跳过先验**：诊断前必须读经验库，等于必须先有 prior。裸诊断 = 无先验的后验 = 无效。
- **不得自证**：后验不能由模型自己验证。只有真实成交数据回填才能把后验升级为新的先验。这是 🔁 回路的哲学基础。
- **置信度演化**：假说(§0) → 单验(§1) → 规律(§2) = 先验越来越强。对应贝叶斯的"后验做下一次的先验"。

**与现有自进化回路的对应**：
- 意义闸（命题7）= 贝叶斯框架下的「这个假说能不能被证据检验？」
- §0 待验证隔离区 = 后验等待证据
- 棘轮升降级 = P(H|E) 更新
- 禁止模型自评 = 「证据只能来自现实，不能来自模型自己」

### 6. 斯多葛 · 控制圈边界

**核心问题**：「这堆问题里，哪些是我能控制的？哪些只能观察？」

**来源**：爱比克泰德「有些事在我们掌控之中，有些事不在」。

**怎么用**（激活于 ⑤运营策略收尾 + 💬复诊收尾）：

每条策略/诊断收尾，强制标注控制圈：

| 可控（行动域） | 不可控（观察域） |
|---------------|----------------|
| 主图/详情页/视觉 | 平台算法规则变化 |
| 投放预算分配 | 竞品定价策略 |
| 客服话术/响应速度 | 大环境消费力 |
| 评论维护节奏 | 竞品投放力度 |
| 活动报名/节奏 | 大盘流量波动 |

**铁律**：
- **可控的 → 立即给行动清单**（优先级排序 + 时间窗）。
- **不可控的 → 只设观察哨**（"如果 X 变化超过阈值 Y，则触发 Z 预案"），不输出焦虑型判断（如"平台流量下降你完了"——你控不了的事不值得焦虑）。
- **灰色地带 → 拆**（"DSR 评分" = 客服话术你控 + 物流速度你半控 + 恶意差评你不可控，拆开分别处理）。

> **对应贯穿红线③「不预测业绩」** — 预测本质上是在不可控域里做判断。斯多葛帮你区分：可控域里的判断叫「策略」，不可控域里的判断叫「算命」。
