# 数据分析（电商运营诊断 · 第④段）

> 读店铺真实经营数据，定位**黑洞渠道 / 死流量人群 / 王者 SKU**。
> 铁律：**确定性的部分用算术算（明示阈值，不靠 AI 拍脑袋）；AI 只加定性解读、绝不碰数字、绝不预测业绩。**
> 口径源自工具代码 `metrics/dataAnalysis.ts`（算 flag）+ `ai/dataInsight.ts`（AI 解读）。

---

## 触发

用户给出三类经营数据（生意参谋/店透视等），想知道"这堆数字说明什么"。需要**真实数字**，没有就停下来要，不编。

## 输入：三联表（手填真实数）

1. **流量结构表**：每个渠道（如 店播 / 直通车 / 自然搜索 / 推荐）的 `访客数`、`订单数`。
2. **客群错位表**：每个人群的 `访问占比%`、`支付占比%`（来源标生意参谋"竞店分析-客群分析"等）。
3. **SKU 矩阵表**：每个 SKU 的 `访客数`、`订单数`。

可选：行业转化率基准 `cvrBenchmark`（没有就按店铺自身大盘比）。

---

## 第一步：确定性引擎（先算，阈值写死，不容 AI 改）

转化率 `cvr = 订单数 / 访客数`（百分比，2 位）。大盘转化率 = 总订单 / 总访客。

**渠道 flag**：
- 访客 > 0 且订单 = 0 → **黑洞**（纯烧曝光不出单）
- 有基准：cvr < 基准 → 低效；无基准：cvr < 大盘×50% → 低效
- cvr ≥ 大盘×2 → **高效**
- 其余 → 正常

**客群错位 flag**（gap = 支付占比 − 访问占比）：
- gap ≤ −10pp → **死流量**（来得多、买得少）
- gap ≥ +10pp → **高效转化**（来得少、买得狠）
- 其余 → 匹配

**SKU flag**（以 SKU 池整体转化率为参照，避免小样本被王者自身拖高）：
- 访客 > 0 且订单 = 0 → 僵尸
- cvr ≥ SKU池大盘×3 **且 订单 ≥ 3** → **王者**（够格升主推）
- cvr ≥ SKU池大盘 → 主力
- 其余 → 普通

> 阈值都明示在这。把每行算完、打上 flag，再交给 AI 解读——AI 不准改这些数字和 flag。

---

## 第二步：AI 定性解读（只解读，不算数）

对已打好 flag 的三联表，产出一个 JSON：

```json
{
  "insights": {
    "traffic": ["店播1000访客0转化=纯黑洞，烧曝光不出单"],
    "audience": ["访问端中老年占比高、支付端却是白领 → 投放人群与成交人群错位"],
    "sku": ["SKUx 转化是大盘数倍=真王者，应升主推"]
  },
  "improvements": [
    {"priority": "red", "text": "砍掉0转化的店播渠道预算，转投出单渠道"}
  ]
}
```

**铁律**：
1. **严禁预测业绩数字**（"改完销量+X%""转化率提升X%"一律禁）——只诊断现状 + 给方向。
2. 数字以三联表为准，**不另编数字**。
3. 每维 `insights` 给 1-4 条，针对**具体渠道/人群/SKU 名**，不空话。
4. `improvements` 按优先级（red/yellow/green）给 2-5 条，可执行。
5. 客群错位只说"投放与成交错位"，**不编"X% 的人怎样"**。

---

## 输出

- 三联表（带 flag）+ AI 解读 + 改进清单。
- 末尾附「数据完备度 x/10」与「判断不了的」（如：5 单样本太小，结论需更多成交验证）。
- 哪张表没给 → 该维度标"未做"，不编。

## 往下接

数据分析识别的「客群错位」→ 指导**人群验证**该代入哪些有份量的人群；三联结论摘要 → 喂**运营策略**。
