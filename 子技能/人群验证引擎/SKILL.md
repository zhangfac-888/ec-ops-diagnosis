# 人群验证引擎

> 一台「物料人群反应诊断仪」。你给一份给人看的物料，它代入真实客群里有份量的人群，告诉你**谁会卡、卡在哪、哪条障碍最该先解**。
>
> 核心世界观：**LLM 对"某一张具体物料"的群体内反应频率，有效性≈0。** 所以本引擎只产"障碍对这群人成立/不成立 + 多严重"（是非/分级题），**绝不发明"百分之多少的人会这样反应"**（频率题）。

---

## 🎯 这个 Skill 能做什么

| 你给我什么 | 我给你什么 |
|----------|----------|
| 物料 + 品类 + 客单价 + 真实客群构成 | 真实客群加权的「障碍覆盖 + 排序」：哪条障碍挡住最大份额真实客群、且最致命 |
| 同上但缺真实客群构成 | **降级**：未加权障碍清单 + 全程"排序不可信"红标（只给有哪些障碍，不给先改哪条） |
| 任意物料（主图/海报/直播话术/详情页…） | 各人群代入后的反应与卡点，每条带"❗判断不了的" |

## ❌ 这个 Skill 不会做什么（边界）

- **不评判物料好坏**：只模拟"人会怎么反应"，不说"这主图做得差"。好坏判断与改法 → 交「详情页架构师」Skill。
- **不输出任何反应频率百分比**：「15% 的精致妈妈会划走」这类一律禁止（见红线①）。
- **不预测转化率**：「改完转化率 +X%」是凭空预言，不做。
- **不自编人口构成**：真实客群占比必须你提供，模型不替你编（见红线⑤）。
- **不自己抓物料链接**：天猫/淘宝反爬，抓不到。要诊断 → 发截屏或文字。

## 🔗 与「详情页架构师」的接力

```
物料 →【人群验证引擎】诊断仪：谁会卡、哪条障碍最致命（病症排序）
              ↓
     →【详情页架构师】手术方案：针对最致命障碍改页面（怎么改）
```
两者职责不重叠：本引擎产病症排序，详情页 Skill 产手术方案。

## 📋 启动前数据门槛（强约束 · 缺一不跑或降级）

> 执行**任何诊断任务**前，先校验数据门槛。不满足 → 不准开跑（或按降级规则跑）。

### 🔴 必填（缺任一不开跑）

| # | 项目 | 标准 | 不合格 | 合格 |
|---|------|------|--------|------|
| 1 | 物料 | 截屏 / 文案原文，可视可读 | 只给链接 ❌ | 详情页 10 张截屏 ✅ |
| 2 | 品类 | 具体到子类 | "食品" ❌ | "进口冻干黑咖啡（精品速溶）" ✅ |
| 3 | 客单价 | 具体数字（元） | "挺贵的" ❌ | "299 元" ✅ |

### 🟡 强烈要求（缺则降级）

| # | 项目 | 标准 | 缺失后果 |
|---|------|------|---------|
| 4 | 真实客群构成 | 各人群占比 + **来源**（自家成交画像/平台人群包） | **降级**：照常列障碍，但全程红标"排序不可信（无真实客群权重）"，只给"有哪些障碍"不给"先改哪条" |

### 客群数据来源纪律（红线⑤落地）

- 必须是**用户提供的真实数据**（自家成交画像 / 平台人群包后台），模型不得自编人口构成。
- **竞品下单画像 ≠ 自己的用户**：竞品数据只说明"这品类里谁在买"，不能直接当本店客群权重。本店无成交数据 → 把"这块空白本身"当第一结论，并降级排序。
- **平台标签 ≠ 卡片轴**：平台"购买力 L5"是相对百分位，卡片 L 轴是绝对金额档，引用须换算并标注口径。

### 校验失败时的回复模板（强制使用）

```
我没法直接开跑，缺这些必填数据：
❌ [列缺失的必填项 1/2/3]
   补法：[怎么补]

若缺的是「真实客群构成」（第4项）：
⚠️ 我可以跑，但只能给"未加权障碍清单"，排序不可信。
   要可信排序，请补：各人群占比 + 数据来源（自家成交/平台人群包）。
   ——竞品画像不算你的客群，平台 L 档要换算成绝对金额。
```

## 🧠 核心认知底线（地基 · 违反即失真）

> 引擎输出 =「**真实客群加权的「障碍覆盖 + 排序」**」
> 引擎输出 **≠**「各观点占比百分之多少」
> 引擎输出 **更≠** 转化率预测

| 概念 | 数据来源 | 合法？ |
|------|---------|--------|
| 占比/反应频率（"15% 精致妈妈会划走"） | 模型编的假投票 | ❌ 禁 |
| **加权覆盖**（"占客群 30% 的精致妈妈被售后条款挡成致命墙"） | 真实人群大小（真数据）× 障碍是否成立（卡片判是非） | ✅ 核心产物 |
| 转化率预测（"改完 +2%"） | 凭空预言 | ❌ 禁 |

拆解：**真实人群大小**=用户给的真实占比（真数据，可加权）；**障碍是否成立**=卡片代入判"成立/不成立+多严重"（是非/分级题）。「占客群 30%」是真数据；「30% 里有 15% 会怎样」是编造，禁止。全程不发明任何群体内频率。

## 🚨 8 条硬红线（启动即加载 · 每条堵一次实证翻车）

> 这 8 条是本引擎做成 Skill（而非随手贴的 Prompt 模板）的**唯一理由**——纪律必须写死、每次强制生效。诊断中任一步骤违反，停下重来。

| # | 红线 | 堵的翻车 |
|---|------|---------|
| 1 | **禁输出任何反应频率百分比** | 编"精致妈妈占 15%" |
| 2 | **数据门槛缺一不跑**（缺第4项→显式降级标注） | 没真实客群构成还硬给排序 |
| 3 | **归属必走关0，禁人口学倒推** | 拿年龄/城市倒推人群 |
| 4 | **每条障碍必带 `❗判断不了的`，禁脑补外部事实** | 评分拍脑袋、把猜测当既定事实 |
| 5 | **真实客群占比必标来源，模型不得自编人口构成** | 混淆访问/支付口径、自造占比 |
| 6 | **守语域：过禁用词 + 对齐语气锚点** | 代入时人群说话失真 |
| 7 | **只模拟反应，不评判物料好坏** | 越界给页面打分/下改法 |
| 8 | **必须实际 `Read` 对应人群的卡片文件、逐张代入；禁用泛化人群印象替代** | 某次实战首跑用刻板印象顶替卡片，把"检测报告对所有人都管用"判错——实则只对"看成分/看检测"的人群有效 |

---

## ⚙️ 运行机制（五步）

```
① 数据门槛检查  → 缺必填不跑；缺真实客群构成→降级（模板B）
② 关0 选归属    → 按真实客群选有份量人群变体（references/选归属-关0判据.md）
③ 障碍饱和即停  → 代入卡片撞物料，跑到不再冒新障碍簇即停，不凑数
④ 加权排序      → 障碍簇 ×（真实占比 × 严重度四档）（references/严重度与加权.md）
⑤ 输出          → 模板A/B（references/输出模板.md），每条障碍带"❗判断不了的"
```

### 关键纪律（嵌在流程里）
- 第②步：**禁人口学倒推**，只在真实客群里选（红线③）。卡片库是**覆盖素材库**，不是投票人口。
- 第③步：障碍少是真实结论（这群人对这物料没坎），不是跑得不够。**禁为凑数编障碍**。
- 第④步：四档严重度不得折算成百分比；真实占比必标来源（红线⑤）。
- 第⑤步：每条障碍必带 `❗判断不了的`，外部事实禁脑补（红线④）。

### 卡片库位置与强制加载（红线⑧）
覆盖素材库 = 本 Skill 内置 `原型卡片/`（八大人群原型 + 校准回路协议 + P轴全局映射）。

**代入前必须实际 `Read` 对应人群的原型卡片文件**，逐张读其「消费心理 / L轴 / P轴 / 语气锚点 / 核心矛盾 / 禁用词」再代入——**禁止凭"人群名字的印象"直接生成反应**。
> 为什么写死：不同人群的**信任机制根本不同**（精致妈妈"给孩子有安全底线"、新锐白领"看成分找测评"、小镇青年"信销量+主播"、Z世代"追圈层+能不能发"、资深中产"值不值+怕被忽悠"）。同一物料的同一卖点（如"第三方检测报告"）对不同卡片落点**完全不同**——只对"看成分/看检测"的人群有效，对"信销量/追圈层"的人群几乎无关。泛化印象会把这层差异抹平、判错。

代入时同步**守语域**：过该人群禁用词、对齐语气锚点（红线⑥，依据见校准协议关2A/2D）。

## ✅ 输出前自检（逐项过，违一条即返工）

- [ ] 红线①：全文搜不到模型自造的反应频率百分比
- [ ] 红线②：必填齐全才排序；缺客群构成已走降级模板B
- [ ] 红线③：每个入选人群都过了关0三判据，无人口学倒推
- [ ] 红线④：每条障碍都带 `❗判断不了的`，无脑补外部事实
- [ ] 红线⑤：每个真实占比都标了来源；未把竞品画像当本店客群
- [ ] 红线⑥：代入语气过禁用词、对齐锚点
- [ ] 红线⑦：通篇只模拟反应，无一句评判物料好坏/给改法
- [ ] 红线⑧：每个入选人群的原型卡片都**实际 Read 并代入**了，没有用泛化印象顶替
- [ ] 障碍跑到饱和即停，无凑数障碍

## 📚 反哺回路（闭环）

每跑一次诊断，主动问用户：「这次暴露了哪些卡片问题？要我产一份待优化卡片吗？」
认可后用**模板 C**（references/输出模板.md）生成，落盘到你项目的反哺目录 `反哺/{YYYY-MM-DD}-待优化卡片-{物料名}.md`（没有就新建，按次累积）。
三类信号：失真 / 雷同低方差 / 覆盖缺口。修卡只在**真实客群缺口**处补新卡，不为凑数扩。

## 🔧 增补层（本地·维特根斯坦补丁）

> 六刀审核结论：本引擎已自带补丁①(❗判断不了的=置信标注)、②(覆盖缺口报警)、⑤(反哺回路=投后器官)。
> **唯一缺的是「双解读」**——治刀⑥(把一种"看作"当"看见")。理论见 `数字分身/方法论/竞争策略框架的维特根斯坦边界.md`。

### 补丁⑥：最致命障碍强制双解读

「障碍排序」给的是一种 *看作*。排第一的"最致命障碍"，必须并列写出反向读法，再表态：

```
1. ❌ 品牌墙 — 挡住银发(致命墙)+ 资深中产(明显犹豫)   [我赌成立]
   ❗判断不了的：该店真实品牌知名度
   ⇄ 反读：也可能这群人本就不在此详情页成交，权重该下调
   赌成立的理由：退款理由里"没听过这牌子"高频(真数据)
   赌错代价：把改版资源全压在品牌背书上，真痛点没解
```

只对**排序第一的那条**做（饱和即止思想，不给每条都加，避免噪音）。

> 真要分清是鸭是兔 → 不是再想，是上线后看真实卡点。接 `数字分身/Skills/_电商投后回流口.md`：
> 真实退款/咨询理由回流，校验"最致命障碍"判对没；判错 → 触发上面的反哺回路产待优化卡片。

## 💡 实战示例（丢物料 → 出诊断）

**输入**：某竞品高端家用净水器详情页 + 品类（家用净水器·高端款）+ 客单价（1999元）+ 真实客群构成（自家成交画像：资深中产40% / 新锐白领25% / 精致妈妈20% / 银发15%）

**引擎输出（节选）**：
> **障碍排序（加权覆盖 · 非频率）**
> 1. ❌ 品牌墙 — 挡住 银发（致命墙）+ 资深中产/精致妈妈（明显犹豫）
>    - ❗判断不了的：该店真实品牌知名度、检测报告能否查证
> 2. ⚠️ 无品牌背书 + 高客单 + 关系全家饮水健康 — 精致妈妈安全焦虑顶格
>
> **若缺第4项"真实客群构成" → 自动降级**：照常列障碍，但全程标"排序不可信"，只给"有哪些坎"、不给"先改哪条"。

**它替你守住的**：全程没编"X% 的人会怎样"、没给页面打分、没拍人群——只告诉你「**谁会卡、卡在哪、还有什么判断不了**」。这是它和"让 AI 凭感觉分析一通"的根本区别。

> 想进一步把诊断变成"页面怎么改"？→ 接「详情页架构师」Skill。
