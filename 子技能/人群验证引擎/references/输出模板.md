# 输出模板

## 模板 A：正常诊断（数据齐全）

```markdown
## 诊断对象
[物料一句话] | 品类：__ | 客单价：__ 元
真实客群构成：[各人群占比 + 来源]

## 入选人群变体（关0 选出）
- [人群A 变体]（真实占比 __%，来源：__）
- [人群B 变体]（真实占比 __%，来源：__）
- 略过：[谁 + 为什么占比可忽略]

## 障碍排序（加权覆盖 · 非频率）
1. ❌ [障碍簇] — 挡住 [人群A+B，合计真实占比 __%]，对其=「致命墙」
   - 为什么成立：[卡片逻辑，引该人群消费叙事/信任来源]
   - ❗判断不了的：[需真实销量/评价/竞品价等外部事实才能确认的部分]
2. ⚠️ [障碍簇] — 挡住 [人群C，__%]，=「明显犹豫」
   - 为什么成立：…
   - ❗判断不了的：…
（到障碍饱和即停，不凑数）

## 排序可信度
✅ 已提供真实客群构成 → 排序可信
```

## 模板 B：降级诊断（缺真实客群构成 · 红线②落地）

```markdown
## 诊断对象
[物料一句话] | 品类：__ | 客单价：__ 元
真实客群构成：⛔ 未提供 → 本次排序不可信

## 🔴 排序不可信声明（必置顶）
未拿到真实客群构成，无法加权。以下只列"有哪些障碍"，**不给"先改哪条"**。
要可信排序，请补：各人群占比 + 数据来源（自家成交/平台人群包）。
注意：竞品画像不是你的客群；平台 L 档需换算成绝对金额。

## 未加权障碍清单（不排序）
- ❌/⚠️/➖ [障碍簇] — 可能影响 [推测人群]，严重度 [档]
  - 为什么成立：…
  - ❗判断不了的：…
（按发现顺序列，不按重要性排）
```

## 模板 C：待优化卡片（反哺产物）

> 落盘：你项目的反哺目录 `反哺/{YYYY-MM-DD}-待优化卡片-{物料名}.md`（没有就新建），按次累积，不覆盖。

```markdown
# 待优化卡片 — [物料名] — [日期]

> 由人群验证引擎运行副产。三类信号 → 喂回卡片库修卡。

## 失真信号（卡片说话不像这群人 / 逻辑不成立）
- [人群/变体]：[哪句失真] → 修法：[对照校准协议关2，禁用词/语域/核心矛盾]

## 雷同低方差信号（几张卡撞同物料反应几乎一样）
- [人群A vs 人群B]：[撞在哪] → 查近邻盲测，强化差异轴

## 覆盖缺口信号（真实客群有一类，卡库无对应变体）
- [缺的人群/变体]：[真实客群里的依据] → 只在此真实缺口处补新卡（不为凑数扩）
```
