# 详情页架构师 · 方法论手册

> 本文档是 `详情页架构师` Skill 的方法论参考。SKILL.md 是 AI 执行指令，本文档是底层原理说明。
>
> 三个原则：**用户决策驱动**、**数据来源透明**、**结构可复用**。

---

## 一、详情页的底层认知

### 详情页不是商品图册，是决策辅助工具

用户点进详情页时不在看产品，**在替自己回答一个问题**：

> 「我该不该花这笔钱、买这个、用来解决我的某个问题？」

详情页的每一屏，都要服务于这个决策。任何不推进决策的屏，都是浪费。

### 用户购物的四阶段决策路径

```
1. 产生需求
   主图/标题让用户产生兴趣
   ↓
2. 开始行动
   点进详情页开始了解
   ↓
3. 进行比较
   对比竞品，寻找购买理由 / 排除疑虑
   ↓
4. 下单购买
   消除最后顾虑，完成转化
```

12-15 屏的详情页本质上是把这条决策路径**展开成屏**：
- 第 1 屏 = 阶段 1 的延续（强化兴趣）
- 第 2-8 屏 = 阶段 2-3（提供比较材料）
- 第 9-15 屏 = 阶段 4（消除顾虑 + 行动号召）

写不出某一屏要表达什么，就回到「这一屏服务于哪个决策阶段」。

## 二、卖点挖掘四维法

详情页要写什么内容，不能拍脑袋决定。必须从四个数据源挖掘。

> 注意：四维法**依赖外部数据**。AI 在 SKILL.md 「启动前数据契约」里强制要求用户提供必填项，原因就是没有数据就只能凭空编。

### 维度一：对手对比（找差异化优势）

**核心问题**：竞品在打什么牌？我能打他们没打过的什么牌？

**操作动作**：
1. 列出销量前 5 的竞品（用户提供）
2. 逐个分析他们详情页前 5 屏的核心卖点
3. 把他们的卖点列出来对照
4. 找出**他们没强调但你也有**的能力——这就是你的差异化优势

**案例**：胶囊咖啡机

| 竞品 | 主推卖点 |
|------|---------|
| N 品牌 | 咖啡馆品质（与连锁咖啡馆联名） |
| D 品牌 | 无线便携（户外/办公便携） |
| I 品牌 | 性价比（同档低 30%） |

发现的空白点：**没有一个竞品强调「30 秒出杯不等机器醒」**——而这恰恰是上班族晨间最痛的体验。
→ 把「30 秒出杯」作为第一屏核心卖点。

### 维度二：搜索词分析（找用户最关心的点）

**核心问题**：用户搜索时附加了什么修饰词？这些修饰词暴露了什么担忧？

**数据来源**：生意参谋的搜索词报告

**操作动作**：
1. 用主词搜索，看用户高频附加的修饰词
2. 修饰词暴露了用户的隐藏担忧
3. 在详情页明确回应这些担忧

**案例**：自动反向伞

搜索"反向伞"的用户还附加搜索：
- **"加大"** → 担心伞面不够大不能护到衣服
- **"防晒"** → 不只防雨，也防晒
- **"自动"** → 担心收伞麻烦
- **"加固"** → 担心被风吹翻

结论：第 1 屏标题应该是「**加大伞面 / 自动收 / 抗风吹**」，三个关键修饰词都明确回应。

### 维度三：评价与问大家（找痛点与爽点）

**核心问题**：现有用户买完之后是什么感受？好评和差评分别说了什么？

**数据来源**：竞品评价区 / 问大家区

**操作动作**：
1. 抓竞品评价 Top 50（好评 + 差评）
2. **好评高频词 = 你可以强化的卖点**
3. **差评高频词 = 你需要主动解决的问题**
4. 问大家高频问题 = 用户还没被回答的疑虑

**案例**：控油洁面慕斯

好评高频词：「不紧绷」「泡沫细腻」「温和不刺激」
差评高频词：「味道太香」「瓶口出泡量难控制」「用完干」

对策：
- **第 2 屏强化**：「氨基酸表活 + 弱酸 pH5.5，洗完不紧绷」（呼应好评）
- **第 6 屏主动解决**：「淡香型香氛 / 0.5 秒按压精准控量」（化解差评）
- **第 11 屏 FAQ**：「敏感肌可用吗？已通过 30 人封闭斑贴测试」（回应问大家）

### 维度四：产品属性罗列（找技术支撑）

**核心问题**：你产品参数表里的每一项，用户能看懂吗？能产生具体感受吗？

**操作动作**：
1. 列出所有产品参数（功能 / 材质 / 规格 / 工艺 / 认证）
2. 把每个参数用「属性 + 解决方案」公式翻译成用户语言
3. 翻译时强调「这个参数让你具体得到了什么体验」

**案例**：胶囊咖啡机参数翻译

| 参数（产品语言） | 翻译（用户语言） |
|----------------|----------------|
| 19 Bar 高压萃取 | 高压萃取，能压出连锁咖啡馆同款的「克丽玛」奶油层 |
| 0.6L 水箱可拆洗 | 水箱拆下来直接冲，不留水垢 |
| 30 秒出杯 | 起床洗漱完，咖啡已经在杯里 |
| 19 dB 静音 | 早晨用不会吵醒室友 |

**翻译原则**：用户语言 = 能在脑子里成像的体验，不是抽象参数。

## 三、卖点排序：优先级原则

### 三档分类法

**第一档：差异卖点（人无我有 / 人有我优）**
- 判断标准：缺了它用户会去买别人的；加上它显著拉高购买意愿
- 展示位置：**第 1-4 屏**（用户决策初期的关键期）
- 案例：「30 秒出杯不等机器醒」（咖啡机）

**第二档：常见卖点（行业基本门槛）**
- 判断标准：用户默认你该有的功能，没有就劝退，但有也不算亮点
- 展示位置：**第 5-8 屏**
- 案例：「19 Bar 高压萃取」「兼容主流胶囊」（咖啡机的行业标配）

**第三档：购买保障（消除最后顾虑）**
- 判断标准：用户不会主动看，但看不到会犹豫
- 展示位置：**第 9-12 屏**
- 案例：「7 天无理由退换」「2 年质保」「运费险已投保」

### 排序判断的两个问题

每个卖点放进详情页前，问自己：

1. **「如果缺了这个卖点，用户会放弃购买吗？」**
   - 是 → 它是关键卖点（差异或常见）
   - 否 → 它顶多是加分项，别放前 8 屏
2. **「加入这个卖点，用户购买意愿是否显著提升？」**
   - 是 → 放前 3 屏
   - 否 → 放中后段或不放

两个问题都"否"的卖点 → **删掉**，不要堆字数。

## 四、12-15 屏黄金结构

### 总分布

| 屏号 | 任务 | 时长权重 |
|------|------|---------|
| 第 1 屏 | 黄金屏：抓住注意力，建立差异化认知 | 50% 决策由此屏决定 |
| 第 2-4 屏 | 围绕第 1 屏卖点展开（原理 / 场景 / 对比） | 加深兴趣 |
| 第 5-8 屏 | 常见卖点和功能详解 | 完成基本信息传递 |
| 第 9-12 屏 | 购买保障 + 用户见证 | 消除顾虑 |
| 第 13-15 屏 | FAQ + 行动号召 | 临门一脚 |

### 第 1 屏（黄金屏）的强制结构

```
主标题（差异卖点 + 信任状）
+ 主图
+ 4 个证明点
+ 促购引导
```

**示例：胶囊咖啡机第 1 屏**

```
主标题：
30 秒出杯  起床就喝
（差异卖点）

副标题：
19 Bar 高压萃取 · 国标认证家用咖啡机
（信任状）

4 个证明点：
✓ 累计卖出 38000+ 台
✓ 京东自营好评率 98.6%
✓ 通过 3C 认证 + 国家家电检测
✓ 30 天无理由 + 2 年质保

促购引导：
限时立减 100 元，前 50 名加赠胶囊礼盒
```

### 第 2-4 屏：围绕差异卖点展开

**模板**：
- 第 2 屏 = 技术原理（为什么你能做到）
- 第 3 屏 = 使用场景（什么时候用）
- 第 4 屏 = 对比优势（比别人好在哪）

**示例：咖啡机的「30 秒出杯」三屏**

```
第 2 屏：技术原理
「3 秒预热 + 27 秒萃取 = 30 秒出杯」
配图：内部结构剖面图 + 加热曲线对比

第 3 屏：使用场景
「7:30 起床 → 7:35 咖啡入杯 → 7:40 出门」
配图：晨间作息时间轴

第 4 屏：对比优势
「同价位竞品平均出杯时间 80 秒，慢 2.6 倍」
配图：4 个品牌出杯时间柱状图
```

### 第 5-8 屏：常见卖点详解

按重要性排列：核心功能 → 材质工艺 → 易用性 → 多场景适配。

每个功能用「属性 + 解决方案」公式表达，不要堆参数表。

### 第 9-12 屏：购买保障

```
第 9 屏：用户见证（带图好评精选）
第 10 屏：售后政策（退换 / 质保 / 安装）
第 11 屏：FAQ（5-8 个高频问题）
第 12 屏：物流说明（仓库 / 时效 / 包装）
```

### 第 13-15 屏：临门一脚

```
第 13 屏：限时优惠倒计时
第 14 屏：库存紧张提示（如适用）
第 15 屏：再次强调核心卖点 + 立即购买按钮
```

## 五、因果文案公式

### 核心公式

**产品属性 + 解决方案（因果逻辑）**

属性是因，解决方案是果。两者之间要有清晰的因果链。

### 错误示范 vs 正确示范

| ❌ 错误（空洞描述） | ✅ 正确（属性导出体验） |
|-----|-----|
| 「我们的咖啡机很好用」 | 「30 秒出杯，起床洗漱完咖啡已经在杯里」 |
| 「面料柔软亲肤」 | 「60 支长绒棉，像被云朵包裹」 |
| 「防晒效果好」 | 「UPF50+ 阻隔 98% 紫外线，海边玩 3 小时不晒红」 |
| 「容量大」 | 「0.6L 水箱够泡 6 杯，一周加一次水」 |
| 「质量好」 | 「304 不锈钢内胆 + 3 年质保，过保也帮修」 |

### 文案三技巧

**1. 用具体数字 > 模糊形容**
- ❌ 「很多人买」
- ✅ 「累计卖出 38000+ 台」

**2. 用「极端表达」放大记忆点**
- ❌ 「比较快」
- ✅ 「30 秒出杯，比烧热水还快」

**3. 标题 ≤ 8 字（最多 2 行）**
- ❌ 「采用先进高压萃取技术的家用咖啡机」
- ✅ 「19 Bar 高压」换行「咖啡馆口感」

### 高风险词清单（合规约束）

电商详情页禁止使用绝对化表述。以下词慎用：

- 「永久」「最」「最好」「第一」「100%」
- 「专用」「特效」「绝对」
- 「治疗」「治愈」「彻底」（医疗类强禁）
- 「环保」「认证」「食品级」（无相应认证时禁用）
- 「承重 XX 斤」（无第三方测试报告时禁用）

广告法的红线，违反会被平台处罚。AI 在写文案时要主动避开。

## 六、视觉表达规范

### 用户视线轨迹：F 型模型

用户浏览详情页时眼睛走的是 F 形路径：

```
重要信息 ────────────────→
│
│
重要信息 ────→
│
│
（重要性递减）
```

→ 关键信息放**左侧和顶部**。
→ 图片必须精准传达文案含义，不要"图很美但与文案无关"。

### 背景色规范

- **单板块**：≤ 3 种颜色（避免视觉混乱）
- **整页**：最多 2 种背景色交替（建立节奏）
- **禁忌大面积背景色**：红（廉价感）、灰（沉闷）、绿（食品类除外）

### 推荐配色方法

1. **从产品本身提色**（咖啡机本体深棕色 → 详情页用米白 + 深棕）
2. **使用功能象征色**（蓝色 = 凉爽 / 紫色 = 高端 / 橙色 = 活力）
3. **极简纯色**（白底 + 一种主色调）

### 字体规范

| 用途 | 推荐字体 |
|------|---------|
| 主标题 | 粗体 / 艺术体（吸引注意） |
| 副标题 | 中粗非衬线体 |
| 正文 | 非衬线体（易读性） |
| 价格/促销 | 粗体红色（紧迫感） |
| 参数表 | 等宽字体（整齐感） |

**艺术体仅用于标题**，不要全屏都艺术体。

### 图片类型选择

| 产品类型 | 推荐图片类型 |
|---------|------------|
| 服饰 | 模特图（展示上身效果） |
| 美妆/护肤 | 模特图 + 用户实拍图 |
| 功能性家电 | 场景图（展示使用场景） |
| 标品/数码 | 白底图（突出产品细节） |
| 食品 | 微距图（展示质感） |

## 七、审查清单（10 分钟过完）

### 内容结构

- [ ] 第一屏包含：主标题 + 配图 + 4 证明点 + 促购引导
- [ ] 主标题符合「差异卖点 + 信任状」公式
- [ ] 前 3-4 屏围绕第一屏的卖点展开
- [ ] 卖点排序：差异 → 常见 → 保障
- [ ] 总屏数 12-15 屏（低客单 6-8 屏除外）

### 文案

- [ ] 每屏标题 ≤ 8 字（或最多 2 行）
- [ ] 使用「属性 + 解决方案」公式
- [ ] 有具体数字支撑（避免「很多」「非常」）
- [ ] 没有违反广告法的绝对化用语
- [ ] 有「新奇」或「极端」表达增强记忆

### 视觉

- [ ] 每屏都有对应的配图要求
- [ ] 图片类型合理（白底图/场景图/模特图）
- [ ] 背景色符合规范（避免大面积红/灰/绿）
- [ ] 字体匹配产品调性

### 转化机制

- [ ] 有限时限量提示
- [ ] 优惠券规则清晰（金额 + 期限 + 叠加）
- [ ] 设置 FAQ 减少客服咨询
- [ ] 购买保障充分（退换 / 质保 / 物流）

## 八、不同品类的适配建议

### 低客单（< 50 元）

- 简化结构 6-8 屏
- 主图 + 标题就能促成转化
- 重点突出性价比
- 不需要复杂的信任状
- 例：洁面慕斯 9.9 元试用装

### 高客单（> 200 元）

- 完整 12-15 屏
- 信任状要厚（专利、认证、媒体报道、KOL 推荐）
- 增加用户实拍 + 第三方评测
- 犹豫周期长，FAQ 要详尽
- 例：胶囊咖啡机 699 元

### 服饰

- 模特图必须有
- 强调面料、版型、穿搭场景
- 提供尺码建议表 + 试穿报告
- 多角度 + 不同身材展示

### 功能性家电/工具

- 强化功能参数标注（功率、容量、认证编号）
- 场景化视觉呈现
- 优先级：**安全性 > 功能性 > 颜值 > 易用性 > 性价比**
- 一定要标注质保和售后

### 美妆个护

- 强调成分（成分表清晰可见）
- 使用感受 + 效果对比图
- 用户实拍图 > 模特图（信任更强）
- 必须标注「个体差异」「敏感肌测试」

### 食品

- 微距图展示质感（咬开拉丝、爆汁等）
- 强调原料产地 + 工艺
- 必须标注保质期 + 储存方式 + 配料表
- 没有 SC 认证不允许做食品

## 九、案例库

### 案例 A：胶囊咖啡机（成功重构）

**原详情页问题**：
- 28 屏，严重超标（用户在第 10 屏左右就失去耐心）
- 前 6 屏全在讲品牌故事
- 第 1 屏没有证明点

**重构方案**：
1. 精简到 14 屏
2. 重构第 1 屏：
   - 主标题：「30 秒出杯  起床就喝」
   - 4 个证明：销量 38000+ / 好评 98.6% / 3C 认证 / 30 天无理由
   - 促购：限时立减 100 元
3. 前 3 屏聚焦「30 秒出杯」核心差异点
4. 中段补强常见卖点
5. 后段加 FAQ + 用户见证

**结果**：客户实测点击-加购转化率提升明显（具体数字需用户实测，方法论不预测）。

### 案例 B：自动反向伞（差异化挖掘）

**调研发现**：
- 搜索词分析：「加大」「防晒」「自动」「加固」是高频修饰词
- 评价差评高频：「伞面太小淋到肩膀」「自动收伞断了」

**详情页对策**：
- 第 1 屏标题：「**伞面加大 30% / 全自动开收 / 抗 7 级风**」
- 第 4 屏：实测视频，吹风机吹伞面对比
- 第 7 屏：拆解图展示「钢化骨架 + 双层叠面」结构

**核心差异化**：从「便宜的伞」变成「专业抗风遮阳伞」。

### 案例 C：控油洁面慕斯（差评驱动设计）

**调研发现**：
- 好评高频：「不紧绷」「泡沫细腻」「温和」
- 差评高频：「太香」「按压泡量难控」「用完干」

**详情页对策**：
- 第 2 屏：氨基酸表活 + pH5.5 = 洗完不紧绷（强化好评点）
- 第 6 屏：「淡香型香氛 / 0.5 秒精准控量」（直接化解前两个差评）
- 第 11 屏 FAQ 第一个问题：「敏感肌可用吗？已通过 30 人封闭斑贴测试」
- 加分项：用户实拍前后对比图

**核心策略**：好评强化 + 差评主动化解 = 信任建立。

## 十、迭代与复盘

### 详情页不是写完就完

每个详情页上线后必须做：

```
1. 数据观察期（7-14 天）
   关键指标：点击-加购率、加购-下单率、详情页停留时长

2. 找出问题屏
   - 跳出率最高的屏 → 哪一屏让用户失去耐心
   - 加购转化最高的屏 → 哪个卖点最打动用户

3. A/B 测试
   每次只改一个变量（标题 / 配图 / 卖点顺序）
   不要一次改 5 屏，无法定位是哪个改动起效

4. 沉淀到经验库
   把"对这个品类有效的卖点 / 失败的表达"写到经验库
   下次做同类品类直接复用
```

### 经验库的真正价值

详情页方法论的天花板，不在「我会不会写」，而在「我有多少品类的真实成功案例」。

Skill 的经验库 (`data/经验库.md`) 是用来积累这些案例的：
- §2 品类卖点库：累积每个品类的高转化卖点
- §3 优秀案例：用户认可的成功案例
- §4 失败教训：用户指出的 AI 做错的地方
- §5 通用规律：跨品类反复验证的洞察

**没有数据回流，方法论永远停留在纸面**。这是用户必须做的工作，AI 替代不了。

## 持续学习路径

1. **基础阶段**：跟着方法论模板做，做出合格的详情页
2. **进阶阶段**：每次上线做 A/B 测试，把结果回流到经验库
3. **高手阶段**：形成自己的品类专属打法，方法论不再是别人的，是自己的

详情页能力的增长不是「学会一套方法」，是「用方法论积累自己的案例库」。
