# 详情页架构师

> 一个用方法论压住 AI 编造倾向的电商详情页工具。
>
> 核心约束：**没有数据不开工**。AI 不会假装替你做你应该做的市场调研。
> 它做的事情：把你给的真实数据，按方法论结构化输出成详情页方案。
>
> **但在所有方法论之上，先读下面这条总则——它决定你怎么用下面的一切。**

---

## ⚖️ 总则：判断优先于规则（读任何其它章节前，先读这条）

> 本 skill 下面写的全是**规则**（契约、四维、排序、12-15 屏、输出模板…）。规则的通病是被**直接执行**，跳过"它适不适用于眼前这一个 case"——这就是套模板。解法不是再加规则，是在规则之上立一层**判断**。

**判断 ≠ 规则。** 规则可复用、可执行；判断是对**当下这一个 case 的一次性裁决**，不可复用、不可固化。判断的产物是"哪几条规则适用、怎么调"，它本身不是又一条规则。

**每次开工的顺序：先裁决，再执行。**
1. **先判后用**：先裁"这个 case 是什么、哪些规则适用/不适用/怎么改"，再决定执行哪些。不是一上来就跑流程、套模板。
2. **规则都带适用条件**：下面每条规则/模板用之前，先问"它的适用条件这个 case 满足吗"，不满足就别用或改用。
3. **经验 = 旧裁决，不是规则**：经验库每条都是过去对某个 case 的一次性裁决，调出来**必须对当下 case 重新裁决，不准 replay**（这就是 §5 规律4 那个 bug 的根）。

**裁决四问**（在 thinking 里逐条作答，不许默认为"是"）：
- ① 这个 case 核心诉求一句话？（不许复述用户原话）
- ② 我要用的规则/模板适配吗？哪里不适配？
- ③ 有什么特殊性是规则没覆盖的？
- ④ 不套规则我会怎么做？
- 结论：`[规则适配:是]` 才照用；`[规则适配:否/部分·因为…]` 以 ④ 为基础自建/改。

**⛔ 判断只管"权变规则"，无权碰"底线"。** 下面这些是 invariant，不论什么 case 都**不准用"判断它不适用"绕过**：
- 真实性 / 不编造（缺数据停下来要，不脑补）
- 数据归属诚实（[数据]/[推断]/[已确认] 如实标）
- 广告法红线（不用 最/第一/100%/特效…）
- 不替用户做市场调研、不预测转化率

> "我判断这个 case 数据契约不适用"——这种话禁止出现。判断管"用哪套打法"，不管"要不要诚实"。

**本总则的三条具体落地补丁**见下文「🔧 判断层·落地补丁」：① 卖点排序双解读（看作≠看见）② 综合评分挂置信（判断≠测量）③ 经验库≠投后回流——它们是维特根斯坦"规则遵循"在本 skill 的具体执行，与本总则同根。

**一句诚实的**：这层判断是**思考姿态，不是机关**。skill 文本只能反复提醒、把规则写成带条件的，挡不住我跳过它；真要硬拦，得靠 skill 之外的独立校验 / 人来查（见经验库 §5 规律7）。

## ❌ 这个 Skill 不会做什么（必须知道的边界）

- **不会自己抓详情页链接**：天猫/淘宝/京东全部有反爬+登录墙，Claude 抓不到内容，硬要"抓"只会基于 URL 编造产品信息。要诊断详情页 → 发截屏或文字。
- **不会预测转化率提升百分比**：LLM 没有数据支撑做不到这件事，凡是说能预测的都在骗你。
- **不会替你做市场调研**：四维分析需要的数据（搜索词、评价、竞品）必须你提供，AI 编造的数据没价值。

## 📋 启动前数据契约（强约束 · 不满足不开工）

> 这是 Skill 的核心硬约束。Claude 在执行**详情页生成任务**前，必须先完成数据契约校验。
> 校验不通过 → 不准开始生成，必须先要数据。

### 🧭 第 0 步：框定产物类型（强制 · 在数据契约之前跑）

> 数据契约只管"有没有数据"，管不了"你在做什么、该用什么方法"。本 skill 的高发翻车不在缺数据，在**做着做着把产物类型搞混了**——把详情页做成主图、把主图当详情页讲因果。所以开工/换任务前先框定：

1. **本次产物是哪种？**（主图 / 详情页 / 文案审查 / 诊断）先说出口，再动手。
2. **加载它对应的方法，并显式声明"它不是另一种"：**
   - **主图 = 钩子**。搜索流/缩略图、0.5 秒、做减法、只留一个**自明或可视**的点。海报，不是论证。判据：缩略图缩到指甲盖还活着。
   - **详情页 = 论证**。用户已点进、阅读态、做加法。图解/对比/报告原件/场景图/成段正文，**一屏杀一个疑虑**。不是把几张主图堆起来。
   - 需要"解释才懂"的因果（如滤芯级数→出水更净）→ 详情页的活，主图放不了（§5 规律5）。
3. **切换产物类型时，强制重置格式**——上一种产物的模板（如主图的"标题+图标+价格"海报）不准带进下一种。
4. **回扫上下文已确认结论**（人群/场景/客单价/信任状/竞品打法）带入本次，别凭惯性从头重来。

> 这是「总则·裁决」的第一个具体动作：先裁出"主图还是详情页"，再按总则「裁决四问」判断该用哪套规则。

⚠️ 凡是"换了任务类型却沿用上一种产物的格式/脑子"——先停，回到第 0 步。这是本 skill 反复犯的同一家族 bug（经验库 §4 / §5 规律7）。

### ❌ 套模板反例库（真实翻车 · 建立"别这么干"的负向边界）

> 只给正面模板＝暗示"照抄就对"。这里钉几个本 skill 真实翻过的车，作为负向边界：

| case | 套模板出了什么 | 为什么错（诉求 vs 模板预设） |
|---|---|---|
| 高端礼盒线挖卖点 | 直接套了"自用买家"的痛点（品质/真假/性价比） | 诉求是「送礼」，模板预设是「自用」——人群×场景不同，痛点不通用（§5 规律4） |
| "做详情页前3屏" | 出成"大标题+图标行+价格"的主图海报 | 诉求是「论证(阅读态/图解/报告原件)」，套成了「主图(0.5秒钩子)」格式（§5 规律7） |
| 详情页第1屏放促购 | 把"促购引导"格塞成"到手价¥XX9"裸价格 | 模板要的是「限时/赠品钩子」，且送礼线价格=暗器；降级成裸价格、又没被策略盖掉（§5 规律8） |
| 自我介绍 | 详情页写"新牌子，凭报告说话" | 自曝弱点＝主动种"无名牌=有风险"的疑（§5 规律5 种疑陷阱） |

### 为什么要数据契约

详情页方法论的"四维分析法"依赖外部数据（竞品/搜索词/评价）。
没有这些数据，AI 只能凭通用品类常识编造，输出就是"看起来对但其实没数据支撑"的纸面分析。
强制契约的目的：让 AI 在**数据缺失时停下来**，不允许靠编填补。

### 🔴 必填项（缺一不开工）

| # | 项目 | 标准 | 不合格示例 | 合格示例 |
|---|------|------|-----------|---------|
| 1 | 产品品类 | 具体到子类，能识别使用场景 | "小家电" ❌ | "胶囊咖啡机（家用单杯/支持奶泡）" ✅ |
| 2 | 客单价 | 具体数字（元） | "便宜的" ❌ | "699 元" ✅ |
| 3 | 目标人群 + 购买场景 | 年龄+城市+身份 ＋ **购买动机（自用 / 送礼 / 囤货…）＋ 核心情境**。痛点是「人群×场景」的属性，场景不明=不准取经验库痛点 | "上班族" ❌ / "送礼的" ❌（没说送谁、什么场合） | "35-45 岁企业中层，买来**送客户**，重体面、怕出错" ✅ |
| 4 | 核心参数 | ≥3 条具体规格/功能/材质 | "操作简单" ❌ | "19 Bar 高压萃取 / 30 秒出杯 / 0.6L 水箱可拆洗" ✅ |
| 5 | 竞品名录 | ≥3 个竞品名字 + 每个的主推卖点 | "市面上有很多" ❌ | "A 品牌主推咖啡馆品质；B 主推无线便携；C 主推性价比" ✅ |

### 🟡 强烈建议（缺了用 [推断] 填，报告会标"数据完备度"下降）

| # | 项目 | 标准 |
|---|------|------|
| 6 | 评价高频词 | 好评 + 差评各 3 个 |
| 7 | 用户搜索修饰词 | 主词后常加的 3-5 个修饰词 |

### 🟢 加分项（有则报告质量明显提升）

| # | 项目 |
|---|------|
| 8 | 生意参谋搜索词导出文本 |
| 9 | 评价区 30+ 条截图或文本 |
| 10 | 问大家高频问题 |

### 契约校验流程

```
用户说"帮我写一个 XX 详情页"
        ↓
Claude 出示「数据契约表」
        ↓
用户提供信息
        ↓
Claude 逐项校验必填 5 项
        ↓
   ┌────┴────┐
   ↓         ↓
有缺漏      全齐
   ↓         ↓
明确指出      校验强烈建议项
缺哪几项      ↓
不准开始    缺的标 [推断]
要数据      ↓
            开始生成 + 输出「数据完备度」分
```

### 校验失败时 Claude 的回复模板（强制使用）

```
我没法直接开始，缺这些必填数据：

❌ [列出缺失的必填项]
   缺失原因：[这一项的标准是什么]
   补法：[告诉用户怎么补]

数据齐全后我才会动笔。
方法论本身依赖数据驱动，缺数据硬填会出错。
```

### 数据质量门控（防止用户敷衍）

收到用户回复后，Claude 还要检查：

- **太宽泛**：人群=「上班族」→ 反问「具体什么年龄段、什么城市、什么使用场景？」
- **自相矛盾**：价格 = 99 元 + 目标人群 = 商务高端 → 反问「客单价和目标人群匹配性可疑」
- **数量不足**：竞品只列 1 个 → 「至少 3 个，否则无法做差异化分析」
- **没指标只情绪**：参数 = 「质量好」→ 「请列具体的功能/规格/材质参数」
- **场景不明**：只说品类不说自用还是送礼 → 反问「自己吃还是送人？送谁、什么场合？」自用与送礼是两拨人，痛点不通用。
- **库里有货 ≠ 适用**：从经验库 §2 调出的痛点/卖点，先核对其 `@人群/@场景` 标签与本单一致；对不上 → 该人群视同「无数据」，不准套用（最危险的不是缺数据，是错人群的数据伪装成完备度，见经验库 §5 规律4）。

只有**全部必填项 + 质量合格**才进入生成阶段。

### 数据完备度评分（每次报告必带）

```
─────────────────────────
📊 本次报告数据完备度：__/10
─────────────────────────
✅ 已用真实数据的章节：[列出]
⚠️ 用 [推断] 填充的章节：[列出]
💡 提升完备度的方法：[告诉用户下次该补什么]

如完备度 < 6：本报告偏向"通用品类指南"，
            决策前建议先把缺失数据补齐再重跑。
```

## 📷 素材获取规则（用于诊断现有详情页）

当用户要诊断**已有**详情页时，必须提供以下任一形式素材：

### A. 截屏（推荐）
从第 1 屏到最后一屏按顺序截图发我。我用视觉模型逐屏审查结构、卖点排序、文案因果公式、视觉规范。
- Windows：`Win + Shift + S`
- Mac：`Cmd + Shift + 4`

### B. 复制粘贴文案
适合纯文案诊断（看不到视觉部分）。

### 禁止行为（强约束）

- ❌ 试图用 WebFetch 抓天猫/淘宝/京东链接（拿到的都是登录页/反爬页）
- ❌ 基于 URL 路径猜产品内容
- ❌ 用通用品类知识硬填诊断
- ❌ 「无法访问，但根据你说的我猜...」这种半编造输出

拿不到素材就明确说**「无法诊断，需要截图或文案」**，等用户提供。

### 例外：小红书 / 微信公众号

用户装了对应 MCP（如 `xiaohongshu-mcp` / `weixin-reader`）时：
- 小红书链接 → 用 MCP 抓
- 微信公众号 → 用 MCP 抓

但仍走详情页诊断方法论。

## 🧠 核心方法论（详见 references/methodology.md）

### 1. 卖点挖掘四维法

每个维度必须标注数据来源（[数据] / [推断] / [用户原话]）：

- **维度一 对手对比**（必须基于用户提供的竞品名录）
- **维度二 搜索词**（用户提供 → [数据]，未提供 → [推断]）
- **维度三 评价痛点**（用户提供 → [数据]，未提供 → [推断]）
- **维度四 产品属性**（基于用户提供的核心参数）

### 2. 卖点排序优先级

```
差异卖点（人无我有 / 人有我优）  → 第 1-4 屏
   ↓
常见卖点（行业基本门槛）         → 第 5-8 屏
   ↓
购买保障（打消顾虑的承诺）       → 第 9-12 屏
```

### 3. 12-15 屏黄金结构

| 屏号 | 内容 |
|------|------|
| 第 1 屏（黄金屏） | 主标题（差异卖点 + 信任状）+ 配图 + 4 个证明点 + 促购引导 |
| 第 2-4 屏 | 围绕第一屏差异卖点展开（技术原理 / 使用场景 / 对比优势） |
| 第 5-8 屏 | 常见卖点与功能详解 |
| 第 9-12 屏 | 购买保障与用户见证 |
| 第 13-15 屏 | FAQ 与行动号召 |

> **适用条件（先判后用）**：12-15 屏只适用于「高客单 + 标准生成」。**不适用**：低客单（< 50 元）走 6-8 屏；只要诊断不要生成；非详情页物料（主图/海报/直播话术）整套不适用。屏数与每屏内容由裁决按 case 定，不是默认填满 15 屏。

### 4. 因果文案公式

**产品属性 + 解决方案**

❌ "面料柔软舒适" → 没有因果，是空洞描述
✅ "60 支长绒棉，像被云朵包裹" → 属性导出体验
✅ "19 Bar 高压萃取，30 秒出杯不等机器醒" → 属性导出场景

每屏标题 ≤ 8 字（超长换行最多 2 行），用具体数字而非模糊形容。

### 5. 视觉规范

- 背景色：单板块 ≤ 3 种，整页最多 2 种交替
- 避免红/灰/绿大面积背景（红色用于促销小标，灰色用于细节图）
- 字体：标题用粗体/艺术体（吸引）；正文用非衬线体（易读）
- 图片类型按品类选：服饰用模特图，功能性用场景图，标品用白底图

## ⚙️ 工作流程（是"裁决后调用的工具序列"，不是自动流水线）

> ⚠️ 读到这里别直接开跑。下面三条流程是**工具**，由「总则·裁决」决定**这个 case 走哪条、跳哪步、怎么改**：低客单可砍屏、只要诊断别启动生成、非详情页物料整条不适用。**先裁决，再选流程。**

### 流程一：从零生成详情页方案（裁决判定"标准生成"时才整条走）

```
1. 数据契约校验
   - 出示契约表
   - 收数据
   - 逐项校验 5 项必填
   - 缺漏则停下要数据（严禁靠常识编造）

2. 卖点挖掘（四维分析，每维标注数据来源）

3. 卖点分类与排序（差异 → 常见 → 保障）

4. 详情页结构规划（12-15 屏内）

5. 文案撰写（因果公式，每屏标题 ≤ 8 字）

6. 配图要求撰写（类型 + 画面 + 背景色 + 字体）

7. 自检清单逐项过

8. 输出数据完备度评分

9. 问用户：「这次有什么经验值得我记到经验库？」
   - 收到反馈用 Edit 工具追加到 data/经验库.md
```

### 流程二：诊断现有详情页

```
1. 收素材（截屏 / 粘贴文案，详见素材获取规则）

2. 按审查清单逐项检查
   - 内容结构（第一屏完整性 / 卖点排序）
   - 卖点有效性（两个问题检验）
   - 视觉呈现（背景色 / 字体 / 图片类型）
   - 文案因果公式（属性+解决方案）
   - 转化机制（限时限量 / 优惠券 / 问答）

3. 输出诊断报告
   - 综合评分 X/100
   - 优势项列表
   - 改进项按优先级（🔴严重 / 🟡中等 / 🟢轻微）
   - 改进项总数（不预测转化率提升幅度）
```

### 流程三：审查/重写现有文案

```
1. 收文案
2. 检查第一屏结构是否完整
3. 验证卖点排序逻辑
4. 检查每句是否符合因果公式
5. 给出问题清单 + 修改示例
```

## 📚 经验库读写规则（强制执行）

> 本 Skill 的"跨会话记忆"通过一个**真实文件**实现，路径：
> `[本 Skill 目录]/data/经验库.md`
>
> 不是模型自动学习，是 Claude 用文件读写模拟出来的"记忆"。
> 只要遵守下面三条规则，第三次合作时确实会越来越懂用户的偏好。

### 规则 1：启动时必读

执行任何详情页任务前，第一步必须：

```
Read [本 Skill 目录]/data/经验库.md
```

读完后（记住总则：**经验是旧裁决、不是规则——调出来必须对当下 case 重新裁决，不准 replay**）：
- 如该品类在 §2 有记录 → **先看痛点条目的 `@人群/@场景` 归属，与本单目标人群/场景一致才用；对不上按"无数据"处理**（痛点是人群×场景的属性、不是品类的属性，见 §5 规律4）
- 如 §1 有用户风格偏好 → 按记录的风格生成
- 如 §3 有相关优秀案例 → 引用作为参考
- 如 §4 有相关失败教训 → 避免重蹈
- 全空 → 走通用方法论，但任务结束后必须写入

### 规则 2：完成时必写

任务结束后主动问用户：

> 「这次有哪些经验值得我记到经验库里？比如：你认可的卖点、不喜欢的表达、要避免的错误。」

收到反馈后，用 **Edit 工具**（不是 Write）追加到对应章节：

| 反馈类型 | 写入位置 |
|---------|---------|
| 用户偏好（风格/字数/严格度） | §1 用户风格偏好 |
| 新品类的卖点/痛点 | §2 品类卖点库 |
| 用户认可的优秀案例 | §3 优秀案例收藏 |
| Claude 这次做错的事 | §4 失败教训 |
| 待回填问题 | §6 待回填问题 |
| 上线后真实数据（转化/各屏停留/退款理由） | §7 投后效果回流（见 `数字分身/Skills/_电商投后回流口.md`） |

写入后告诉用户：「已记录到经验库 §X」

### 规则 3：引用历史时显式说明

应用历史经验时**明示来源**，例如：

> 「根据经验库 §1 你偏好直白风格，这次没用文艺修辞。」
>
> 「经验库 §4 记录过你不接受'极致'这种词，已替换成具体数字。」

引用历史的目的是让用户知道 Skill 真的在用经验库。

## 🎓 不同场景适配

### 低客单价产品（< 50 元）

- 简化结构 6-8 屏
- 重点突出性价比和基础功能
- 主图 + 标题即可促成转化

- 完整 12-15 屏
- 强化信任状（专利、检测报告、媒体背书）

- 必须模特图
- 强调面料、版型、穿搭场景
- 提供尺码建议和试穿报告

- 强化功能参数标注（功率、容量、认证）
- 增加场景化视觉
- 优先级：安全性 > 功能性 > 颜值 > 易用性 > 性价比

### 美妆/个护

- 强调成分 + 使用感受 + 效果对比
- 用户实拍图重要性 > 模特图
- 需明确标注「个体差异」免责

## 🔧 判断层 · 落地补丁（顶部「⚖️ 总则」的具体执行 · 维特根斯坦补丁）

> 这三条是顶部「⚖️ 总则·判断优先于规则」在具体环节的**落地**——维特根斯坦"规则遵循"是总则与这三条补丁**共同的根**。总则讲原则，这里讲在哪几个点上把原则焊死。
> 六刀审核结论：本 skill 已自带补丁①雏形（[数据]/[推断] 标注）、补丁⑤雏形（经验库）。
> 刀③（文案打不打动人）不在此补——由"接力人群验证引擎"在系统层堵。
> 理论见 `数字分身/方法论/竞争策略框架的维特根斯坦边界.md`。

### 补丁③：卖点排序强制双解读（治刀⑥·把"看作"当"看见"）
"差异卖点排第 1-4 屏"是一种 *看作*。进前三屏的那个差异卖点，必须并列写反向读法 + 表态：
```
排第一的差异卖点：全尺码 XL-XXXL   [我赌用户最在乎这个]
⇄ 反读：也可能用户更在乎"价格"，全尺码只是加分项
赌的理由：竞品评论"大码买不到"高频（用户原话）
赌错代价：黄金第一屏喂错卖点，点击/停留全废
```
只对前三屏的差异卖点做，不每屏都加。

### 补丁①强化：综合评分挂置信、禁伪精确（治刀②·评级冒充测量）
诊断的 X/100 是判断不是测量。已在"诊断报告输出结构"焊入：分数后挂「置信·联动完备度」，完备度 < 6 声明仅供方向。永不把 78 分当成量出来的。

### 补丁⑤强化：经验库 ≠ 投后效果回流（治刀⑤残留·用过去假装未来）
现有经验库记的是**你的口味**（偏好/失败教训），不是**市场的反应**。
已加 §7 投后效果回流：上线后真实数据（转化/各屏停留/退款理由）回来，校验当初卖点排序判对没 → 倒改 §2 品类卖点库。
> 接 `数字分身/Skills/_电商投后回流口.md`——那才是"市场说话"的器官，本 skill 的卖点排序在那里被真实数据校验。
